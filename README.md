# Real-wargame

Real-wargame — прототип 2D tactical command game / Soldier Behavior Lab на **Vite + TypeScript + PixiJS**.

Активная работа идёт в ветке:

```text
real-wargame-preview
```

`main` остаётся стабильной базой. Содержательные изменения сначала попадают в `real-wargame-preview`; перенос в `main` делается только после явного GO человека.

## Текущее состояние preview

Этап RTS-заготовки в целом готов. Дальше он считается рабочей основой, которую можно править по ходу разработки поведения солдат.

В preview сейчас есть:

- большая тактическая карта `64×40`, `1 клетка = 10 м`;
- PixiJS-отрисовка карты, объектов, зон, юнитов и приказов;
- зум колесом мыши и перетаскивание карты средней кнопкой или `Space + drag`;
- игровой режим с верхним меню, правыми вкладками и нижней карточкой юнита;
- режим редактора с вкладками, без длинной правой “простыни”;
- кисти высот `-2..+4` и леса `0/1/2`;
- физическая карта высот с кривыми цветными зонами вместо текстовых `+1` на каждой клетке;
- отдельный слой `Реальный рельеф`, который показывает сглаженную высоту для расчёта видимости;
- Alt-линия видимости: зелёная часть видна, красная часть закрыта, расстояния в метрах;
- объекты с физической высотой `losHeightMeters` для линии видимости;
- знания выбранного юнита: ближние укрытия, дальние укрытия для плана, опасность;
- экспорт и загрузка JSON сцены;
- отчёт производительности из браузера;
- GitHub Actions screenshot smoke через Playwright/Chromium;
- data contract AI-графа одиночного солдата;
- headless local AI engine для проверки bundled AI-графа через localhost API;
- AI Node Editor в отдельной вкладке `/ai-node-editor.html`;
- authoring в AI Node Editor: добавить/перетащить/связать/изменить ноду, экспорт/импорт JSON;
- компактный AI Node Editor UI: скрываемые боковые панели, сворачиваемая нижняя консоль, zoom/pan canvas, Fit, контекстное меню и drag-link через порт ноды;
- чистый стартовый canvas AI Node Editor: только `Старт`, без старого демонстрационного дерева поведения;
- простой универсальный набор AI-нод: `Числовой порог`, `Проверка флага`, `Порог расстояния`, `Тактическая проверка`, `Оценка параметра`, `Оценка расстояния`, `Поиск объекта`, `Выбор цели`, `Запись памяти`, `Копия памяти`, `Действие`, `Режим движения`, `Реплика бойца`, `Стабильный порог`, `Запрет действия`;
- универсальные ноды теперь можно связывать цепочкой, например `Проверка флага → Реплика бойца → Поза`;
- старые точечные/legacy-ноды убраны из палитры и local engine, если их можно выразить универсальными нодами;
- общий параметр задержки для нод: `cooldownSeconds` и `cooldownTiming` (`before` / `after`);
- storage AI Node Editor поднят до `v6`, чтобы старый `localStorage` не возвращал прежний грязный canvas;
- AI Game Bridge: игра читает AI-граф из `localStorage v6`, собирает blackboard выбранного бойца и раз в ~0.6 сек исполняет минимальный набор нод прямо в игре;
- видимый goal-срез: `Проверка флага: underFire=true → Реплика бойца → Поза/Действие` уже связан с выбранным бойцом на карте.

Главный смысл проекта на этом этапе — не “красивая RTS”, а удобная лаборатория карты, видимости, укрытий и будущего поведения солдат.

## Пользовательский запуск preview

Для ручной проверки тактической карты использовать:

```text
Run-Real-Wargame.bat
```

Для ручной проверки редактора нод, authoring и local engine использовать:

```text
Run-AI-Node-Editor.bat
```

Для ручной проверки local AI engine stage 2 использовать:

```text
Run-AI-Engine-Smoke.bat
```

Если нужен просто запущенный engine и health JSON в браузере:

```text
Run-AI-Engine.bat
```

Не требовать от пользователя Git-команд, терминала, checkout, merge или ручного переключения веток.

## AI Node Editor stage 4

Текущий AI Node Editor — отдельная вкладка для подпроекта `ai-single-unit-editor`.

Он умеет:

```text
открываться с чистым canvas: только root/Старт;
добавлять универсальные ноды из палитры кнопкой + Add node;
не показывать старые точечные ноды типа Есть приказ / Рядом есть укрытие / Под огнём;
перетаскивать ноды;
перетаскивать canvas за пустое место;
масштабировать canvas колесом мыши и кнопками + / - / Fit;
связывать parent → child протягиванием линии из правой точки-порта ноды;
разрешать цепочки универсальных нод, где проверка может вести к реплике, позе или действию;
открывать контекстное меню правым кликом по ноде;
настраивать универсальную пороговую ноду без JSON: выбрать sourceKey, режим «параметр выше порога» / «параметр ниже порога» и threshold;
настраивать новые универсальные ноды через человеческие панели без JSON на первом уровне;
выбирать фиксированные варианты из списков там, где раньше могли быть псевдо-параметры вроде self;
выставлять задержку каждой ноде: до ноды или после ноды;
делать реплику бойца через ноду «Реплика бойца»;
редактировать displayName/displayNameRu/description/descriptionRu/parameters;
удалять ноды, кроме root;
хранить рабочий граф и позиции в localStorage v6;
экспортировать и импортировать JSON;
проверять изменённый граф через local engine.
```

Подробная ручная проверка:

```text
docs/manual-test/AI_NODE_EDITOR_STAGE_4.md
```

## AI Game Bridge goal-срез

Сейчас игра подключает выбранного бойца к графу из AI Node Editor.

Работает минимальный честный мост:

```text
выбранный боец на карте;
blackboard из текущей игры: danger, stress, underFire, hasOrder, distanceToCover и т.п.;
граф из localStorage v6;
прогон примерно раз в 0.6 секунды;
исполнение FlagCheck / Числовой порог / Порог расстояния / Тактическая проверка / Поиск объекта / Действие / Поза / Режим движения / Реплика бойца;
реплика выводится HTML-плашкой над бойцом;
поза и простое действие записываются в behaviorRuntime бойца.
```

Ограничение: это ещё не полноценный умный солдат и не полный обход всех будущих scoring-нод. Это первый вертикальный мост “нода в редакторе → выбранный боец на карте → видимый результат”.
