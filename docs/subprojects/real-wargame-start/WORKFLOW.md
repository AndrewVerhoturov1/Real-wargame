# Real-wargame workflow: preview-цикл

## Ветки

- `main` — стабильная база. Меняется только после явного разрешения пользователя.
- `real-wargame-preview` — ветка для предварительного просмотра. Все изменения сначала попадают сюда.

## Локальные папки

- Основная: `D:\Codex+opencode_new\Proect_C_O\Real-wargame` — всегда `main`.
- Preview: `D:\Codex+opencode_new\Proect_C_O\Real-wargame-preview` — всегда `real-wargame-preview`.

## Цикл работы

1. Исполнитель (Codex / OpenCode / внешний web-chat) создаёт task-ветку от `main`.
2. Вносит изменения в task-ветке.
3. Переносит изменения в `real-wargame-preview` (через merge/cherry-pick или повторное создание файлов).
4. Открывает (или обновляет) preview-PR из `real-wargame-preview` в `main`.
5. Отчитывается пользователю: что сделано, как тестировать в preview-папке.
6. Пользователь тестирует в `Real-wargame-preview`, запуская `run-preview.bat`.
7. Пользователь говорит: «ГОДИТСЯ» / «GO» — или «ДОРАБОТАТЬ» / «NO-GO».
8. После GO — изменения попадают в `main`.
9. Временная task-ветка закрывается.

## Каждый PR / handoff-отчёт должен содержать

- Какую ветку смотреть.
- Какие файлы изменены.
- Как запустить preview для теста.
- Что проверять и какой результат ожидать.
- Что не проверялось и какие риски.
