# Программа проверки — зум после отчёта производительности v1.8

Цель проверки: снизить подвисания зума, которые отчёт показал не внутри отрисовки карты, а между кадрами браузера. В отчёте средняя отрисовка была быстрой, но были редкие большие провалы кадра. Поэтому в этом пакете снижена постоянная частота тика сцены до 45 FPS, внутренний режим обзора реально выключен по умолчанию, а пустой слой обзора больше не очищается каждый кадр.

## Что открыть

Запускать из папки предпросмотра:

```text
scripts/windows/run-preview.bat
```

Или двойным кликом по корневому файлу:

```text
Run-Real-Wargame.bat
```

## Что проверить руками

1. Проект открывается в браузере.
2. Текст интерфейса по умолчанию русский.
3. Сетка видна по умолчанию.
4. Кнопка сверху должна показывать `Обзор: выкл`.
5. В новом отчёте производительности поле `renderer.viewCones` должно быть `false`, если ты сам не включал обзор.
6. В левом верхнем углу карты видна неподвижная подпись масштаба: `1 клетка = 10 м`.
7. Покрутить колесо мыши для зума 10–20 секунд.
8. Зум должен меньше подвисать, чем в отчёте `v1.7`.
9. Зум не должен возвращаться к длинной плавной анимации через промежуточные кадры.
10. Текст не должен снова начать заметно дёргаться.
11. Подпись масштаба не должна увеличиваться/уменьшаться вместе с картой.
12. Правый интерфейс не должен заметно мерцать или давать эффект “тумана”.
13. Земля и сетка не должны мигать или пересоздаваться визуально.
14. Рыжие зоны не должны мигать или пересоздаваться визуально, если сами зоны не редактируются.
15. Внизу слева нажать плавающую кнопку `Редактор`.
16. После включения редактора игровые разделы должны скрыться.
17. Справа должен остаться отдельный раздел `Инспектор редактора`.
18. В режиме редактора подписи предметов и юнитов на карте должны быть скрыты, чтобы редактор меньше тормозил.
19. В инспекторе редактора должен быть блок `Сохранение`.
20. В блоке `Сохранение` должна быть кнопка `Скачать JSON сцены`.
21. В инспекторе редактора должен быть блок `Отладка производительности`.
22. В блоке `Отладка производительности` должна быть кнопка `Скачать отчёт производительности`.
23. Для проверки отчёта: покрутить зум 10–20 секунд, затем нажать `Скачать отчёт производительности`.
24. Браузер должен скачать файл с именем примерно `real-wargame-performance-...json`.
25. Внутри отчёта должны быть блоки:
    - `browser`;
    - `viewport`;
    - `renderer`;
    - `scene`;
    - `editor`;
    - `summary`;
    - `samples`.
26. В `renderer` должно быть `maxFPS: 45`.
27. В `summary.frameMs.max` желательно не должно быть всплесков 100+ мс при обычном зуме.
28. В `summary.renderMs.avg` отрисовка должна оставаться маленькой.
29. Выбрать инструмент `Выбрать / тянуть`.
30. Кликнуть по предмету на карте: предмет должен подсветиться рамкой.
31. Зажать левую кнопку мыши в центре/на теле предмета и потянуть: предмет должен перемещаться, а не растягиваться.
32. Во время перетаскивания предмета земля и сетка не должны мигать или пересоздаваться визуально.
33. Выбрать инструмент `Создать зону`.
34. Кликнуть по карте: новая зона должна появиться в точке клика и стать выбранной.
35. Выбрать инструмент `Выбрать / тянуть`.
36. Кликнуть по зоне: зона должна подсветиться рамкой/ручками.
37. Зажать левую кнопку мыши внутри зоны и потянуть: зона должна перемещаться.
38. Нажать `Скачать JSON сцены`.
39. Браузер должен скачать файл с именем примерно `real-wargame-scene-...json`.

## Ожидаемый результат

```text
По отчёту v1.7:
- средняя отрисовка была быстрой;
- подвисания были между кадрами браузера;
- значит, надо снижать постоянное давление на браузер, а не только ускорять renderFrame.

Что изменено:
- постоянный тик сцены ограничен до 45 FPS;
- внутренний обзор выключен по умолчанию;
- пустой слой обзора не очищается каждый кадр;
- отчёт теперь пишет maxFPS: 45.

Зум:
- без длинной анимации через requestAnimationFrame;
- шаг зума мягкий;
- величина прокрутки колеса влияет на силу изменения масштаба;
- текст не должен заметно дёргаться.

Отчёт производительности:
- можно скачать JSON-отчёт;
- в отчёте есть FPS, время кадра и время отрисовки;
- в отчёте есть браузер, размер окна, devicePixelRatio;
- в отчёте есть число предметов, юнитов и зон;
- в отчёте есть худшие кадры и худшие отрисовки.

PixiJS:
- канвас PixiJS создаётся без antialias;
- фон канваса непрозрачный;
- зум может выглядеть чуть жёстче, но должен быть легче.

Сохранение:
- редактор может скачать текущую сцену в JSON;
- JSON содержит карту, предметы, юниты и зоны;
- это ещё не автоматическая запись в файлы проекта.
```

## Что не входит в этот пакет

```text
- перевод поля и предметов в Sprite / RenderTexture;
- полностью событийная отрисовка без постоянного тика;
- автоматическая запись JSON прямо в файлы проекта;
- загрузка скачанного JSON обратно через интерфейс;
- перенос симуляции в отдельный рабочий поток;
- физическая коллизия с деревьями, домами, зонами и предметами;
- полноценный поиск пути.
```
