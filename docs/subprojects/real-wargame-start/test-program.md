# Программа проверки — красивый рендер и пачки земли v2.0

Цель проверки: вернуть нормальный внешний вид после отключения сглаживания. Отчёт v1.9 показал, что пачки земли работают: средняя отрисовка маленькая, p95 по отрисовке около 1.6 мс. Поэтому сглаживание PixiJS возвращено обратно, а оптимизация земли пачками сохранена.

## Что открыть

Запускать из папки предпросмотра:

```text
scripts/windows/run-preview.bat
```

Или двойным кликом по корневому файлу:

```text
Run-Real-Wargame.bat
```

## Что проверить руками

1. Проект открывается в браузере.
2. Текст интерфейса по умолчанию русский.
3. Сетка видна по умолчанию.
4. Кнопка сверху должна показывать `Обзор: выкл`.
5. В новом отчёте производительности поле `renderer.viewCones` должно быть `false`, если ты сам не включал обзор.
6. В новом отчёте производительности поле `renderer.maxFPS` должно быть `60`.
7. В новом отчёте производительности поле `renderer.antialias` должно быть `true`.
8. В новом отчёте производительности поле `renderer.mapRender` должно говорить про `batched Pixi Graphics`.
9. Визуально круги, линии зон, рамки и предметы должны выглядеть мягче, чем в варианте с `antialias: false`.
10. В левом верхнем углу карты видна неподвижная подпись масштаба: `1 клетка = 10 м`.
11. Покрутить колесо мыши для зума 10–20 секунд.
12. Зум не должен возвращаться к длинной плавной анимации через промежуточные кадры.
13. Текст не должен снова начать заметно дёргаться.
14. Подпись масштаба не должна увеличиваться/уменьшаться вместе с картой.
15. Правый интерфейс не должен заметно мерцать или давать эффект “тумана”.
16. Земля и сетка не должны мигать или пересоздаваться визуально.
17. Рыжие зоны не должны мигать или пересоздаваться визуально, если сами зоны не редактируются.
18. Внизу слева нажать плавающую кнопку `Редактор`.
19. После включения редактора игровые разделы должны скрыться.
20. Справа должен остаться отдельный раздел `Инспектор редактора`.
21. В режиме редактора подписи предметов и юнитов на карте должны быть скрыты, чтобы редактор меньше тормозил.
22. В инспекторе редактора должен быть блок `Сохранение`.
23. В блоке `Сохранение` должна быть кнопка `Скачать JSON сцены`.
24. В инспекторе редактора должен быть блок `Отладка производительности`.
25. В блоке `Отладка производительности` должна быть кнопка `Скачать отчёт производительности`.
26. Для проверки отчёта: покрутить зум 10–20 секунд, затем нажать `Скачать отчёт производительности`.
27. Браузер должен скачать файл с именем примерно `real-wargame-performance-...json`.
28. Внутри отчёта должны быть блоки:
    - `browser`;
    - `viewport`;
    - `renderer`;
    - `scene`;
    - `editor`;
    - `summary`;
    - `samples`.
29. В `summary.frameMs.p95` желательно должно оставаться около 16–18 мс.
30. В `summary.renderMs.avg` отрисовка должна оставаться маленькой.
31. Выбрать инструмент `Выбрать / тянуть`.
32. Кликнуть по предмету на карте: предмет должен подсветиться рамкой.
33. Зажать левую кнопку мыши в центре/на теле предмета и потянуть: предмет должен перемещаться, а не растягиваться.
34. Во время перетаскивания предмета земля и сетка не должны мигать или пересоздаваться визуально.
35. Выбрать инструмент `Создать зону`.
36. Кликнуть по карте: новая зона должна появиться в точке клика и стать выбранной.
37. Выбрать инструмент `Выбрать / тянуть`.
38. Кликнуть по зоне: зона должна подсветиться рамкой/ручками.
39. Зажать левую кнопку мыши внутри зоны и потянуть: зона должна перемещаться.
40. Нажать `Скачать JSON сцены`.
41. Браузер должен скачать файл с именем примерно `real-wargame-scene-...json`.

## Ожидаемый результат

```text
По отчёту v1.9:
- земля пачками работает лучше, чем 2560 отдельных Graphics-клеток;
- renderMs avg около 1.3 мс;
- renderMs p95 около 1.6 мс;
- значит, можно вернуть сглаживание и проверить внешний вид.

Что изменено:
- antialias снова включён;
- постоянный тик сцены остаётся 60 FPS;
- земля карты остаётся пачками по типам местности;
- вместо 2560 отдельных Graphics-клеток должно быть примерно до 6 больших Graphics-слоёв земли;
- сетка всё ещё отдельный слой;
- предметы и зоны пока остаются отдельными векторными объектами.

Зум:
- без длинной анимации через requestAnimationFrame;
- шаг зума мягкий;
- величина прокрутки колеса влияет на силу изменения масштаба;
- текст не должен заметно дёргаться.

Рендер:
- линии, круги зон и предметы должны выглядеть мягче;
- картинка не должна выглядеть так, будто сглаживания нет.

Отчёт производительности:
- можно скачать JSON-отчёт;
- в отчёте есть FPS, время кадра и время отрисовки;
- в отчёте есть браузер, размер окна, devicePixelRatio;
- в отчёте есть число предметов, юнитов и зон;
- в отчёте есть худшие кадры и худшие отрисовки.

Сохранение:
- редактор может скачать текущую сцену в JSON;
- JSON содержит карту, предметы, юниты и зоны;
- это ещё не автоматическая запись в файлы проекта.
```

## Что не входит в этот пакет

```text
- перевод земли в RenderTexture / Sprite;
- перевод предметов в Sprite;
- полностью событийная отрисовка без постоянного тика;
- автоматическая запись JSON прямо в файлы проекта;
- загрузка скачанного JSON обратно через интерфейс;
- перенос симуляции в отдельный рабочий поток;
- физическая коллизия с деревьями, домами, зонами и предметами;
- полноценный поиск пути.
```
