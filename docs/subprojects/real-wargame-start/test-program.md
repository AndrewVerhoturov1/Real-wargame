# Программа проверки — параметры поведения v0.4

Цель проверки: убедиться, что ветка предпросмотра показывает карту в понятном масштабе, рисует объекты видом сверху, разводит юнитов при совпадении точки назначения и продолжает показывать паспорт солдата без включения полноценного поведения.

## Что открыть

Запускать из папки предпросмотра:

```text
scripts/windows/run-preview.bat
```

Или двойным кликом по корневому файлу:

```text
Run-Real-Wargame.bat
```

## Что проверить руками

1. Проект открывается в браузере.
2. Текст интерфейса по умолчанию русский.
3. Сетка видна по умолчанию.
4. В левом верхнем углу карты видна подпись масштаба: `1 клетка = 10 м`.
5. По сетке понятно, что один квадрат карты равен 10 метрам.
6. Сектор обзора визуально скрыт.
7. На карте видны полупрозрачные рыжие зоны параметров.
8. Предметы на карте выглядят как вид сверху, а не сбоку:
   - деревья — крона сверху с маленьким центром ствола;
   - камни — плоское пятно сверху;
   - дома — прямоугольная крыша сверху;
   - колодец — круг сверху;
   - ящики, брёвна, заборы, мост и канавы — плоские условные знаки сверху.
9. Размеры объектов выглядят соразмерно сетке:
   - обычное дерево меньше одной клетки;
   - дом занимает несколько клеток;
   - камень и ящики заметно меньше клетки;
   - мост, канава и забор растянуты по нескольким клеткам.
10. Правая панель не уходит безвозвратно ниже экрана: её можно прокручивать колесом мыши.
11. В правой панели есть сворачиваемые разделы:
    - `Инспектор поведения`;
    - `Положение юнита`;
    - `Паспорт солдата`.
12. Каждый раздел можно свернуть и развернуть кликом по заголовку.
13. Левым кликом выбирается юнит.
14. В разделе `Инспектор поведения` видны основные параметры выбранного юнита.
15. В разделе `Паспорт солдата` видны постоянные качества, текущие состояния, восприятие и движение.
16. В разделе `Положение юнита` видны кнопки:
    - `▮ Стоя`;
    - `▰ Пригнулся`;
    - `━ Лежит`.
17. Выбрать юнита и нажать каждую кнопку положения.
18. Маркер юнита на карте должен менять вид:
    - стоя — обычный круг;
    - пригнулся — меньший круг;
    - лежит — низкий вытянутый маркер.
19. Отправить одного юнита в точку правым кликом: он должен дойти до точки.
20. Отправить двух юнитов в одну и ту же точку: они не должны полностью слипнуться в один маркер, а должны сохранить небольшое расстояние друг от друга.
21. Проверка коллизии: юниты могут подходить близко, потому что считается только центральная треть тела, но центр одного юнита не должен полностью накладываться на центр другого.
22. Правым кликом дать юниту приказ пройти через зону параметров.
23. Пока юнит реально входит в зону:
    - опасность становится больше 0;
    - стресс растёт;
    - юнит продолжает идти и не застревает;
    - положение не должно насильно менять движение;
    - состояние не должно останавливать движение.
24. Когда юнит выходит из зоны:
    - опасность падает до 0;
    - стресс постепенно снижается.
25. Выбрать разных юнитов и убедиться, что значения паспорта отличаются по профилю.
26. Групповое выделение рамкой продолжает работать.

## Ожидаемый результат

```text
Это ещё не боевое поведение.
Это слой масштаба, отображения и физического места юнитов:
- карта имеет масштаб 1 клетка = 10 м;
- сетка видна и обозначает метры;
- объекты отрисованы как вид сверху;
- размеры объектов заданы в клетках, а не случайными пикселями;
- юниты не слипаются при одной точке назначения;
- коллизия считается только маленьким центральным ядром юнита;
- правая панель прокручивается;
- разделы можно сворачивать;
- паспорт солдата показывает числа;
- зоны считают опасность и стресс;
- настоящее поведение будем задавать позже.
```

## Что не входит в этот пакет

```text
- автоматическое приседание;
- автоматическое положение лёжа;
- остановка от стресса;
- поиск укрытий;
- полноценный поиск пути;
- обход объектов;
- физическая коллизия с деревьями, домами и предметами;
- таблица оценки вариантов действий;
- память и знания юнита;
- полноценный план командира.
```
