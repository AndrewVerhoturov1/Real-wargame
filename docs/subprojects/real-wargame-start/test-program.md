# Программа проверки — RTS-заготовка / Soldier Behavior Lab v0.7

Цель проверки: подтвердить, что текущая RTS-заготовка готова как рабочая основа. Проверяем не финальную игру, а карту, интерфейс, редактор, высоты, лес, укрытия, видимость, физическую высоту объектов и удобство ручного preview.

## Что открыть

Запускать из папки предпросмотра:

```text
scripts/windows/run-preview.bat
```

Пользователю не нужны Git-команды и терминал.

## Базовый экран

1. Игра открывается в браузере.
2. Карта большая: `64×40`, подпись масштаба `1 клетка = 10 м`.
3. Вид по умолчанию чистый:
   - нет постоянной россыпи `+1` / `-1`;
   - view cones выключены;
   - цифры высоты выключены;
   - сглаживание нормальное, без грубого `antialias: false` вида.
4. Зум колесом мыши работает.
5. Перетаскивание карты работает средней кнопкой или `Space + drag`.

## Игровой интерфейс

1. Выбрать солдата.
2. Правая панель должна быть вкладочной, а не длинной простынёй.
3. Вкладки должны быть примерно такими:
   - `Юнит`;
   - `Слои`;
   - `Поведение`;
   - `Сенсоры`.
4. Нижняя карточка юнита должна быть стабильного размера и не прыгать от количества данных.
5. Информация о клетке под мышью должна показывать высоту, тип земли и лес.
6. Расстояния в интерфейсе должны быть в метрах, не в клетках.

## Проверка редактора

1. Нажать верхнюю кнопку режима.
2. Должен открыться редактор как отдельный режим.
3. Редактор должен быть во вкладках:
   - `Инструмент`;
   - `Карта`;
   - `Создать`;
   - `Выбранное`;
   - `Сцена`.
4. Во вкладке `Карта` должны быть кисти высот и леса.
5. Во вкладке `Сцена` должны быть JSON/отчёты/очистка.
6. Проверить создание предмета, юнита и зоны.
7. Проверить движение, размер и поворот предмета.
8. Проверить, что редактор не ломает игровой режим после возврата.

## Проверка высот

1. Высоты должны быть уровнями `-2, -1, 0, +1, +2, +3, +4`.
2. Визуально высоты должны выглядеть как физическая карта: цветные области и кривые границы.
3. Линии высот не должны восприниматься как вертикальные стены.
4. Цифры высот должны включаться только отдельным отладочным переключателем.
5. В инспекторе/интерфейсе должна быть понятна высота клетки под мышью.

## Проверка реального рельефа

1. В верхнем меню нажать `Реальный рельеф: выкл`.
2. Кнопка должна стать `Реальный рельеф: вкл`.
3. На карте должен появиться полупрозрачный слой фактической сглаженной высоты.
4. Этот слой показывает не контурные линии, а реальный слой для расчёта видимости.
5. Слой не должен заметно тормозить при движении мыши: он должен быть кэширован и отделён от подсветки мыши/Alt-линии.

## Проверка леса

1. В редакторе нарисовать редкий лес.
2. Нарисовать густой лес.
3. Вернуться в игру и выбрать солдата.
4. Во вкладке `Слои` ближний лес должен показываться подробнее.
5. Дальний лес должен показываться компактно — как ориентир, а не сотней меток.
6. Ближние известные укрытия должны быть квадратами, дальние ориентиры — кругами.

## Проверка укрытий

1. Выбрать солдата.
2. Открыть вкладку `Слои`.
3. Должны быть отдельные разделы:
   - `Ближние укрытия`;
   - `Укрытия для плана`;
   - `Опасность`.
4. Ближние укрытия должны быть только рядом с юнитом — примерно до 35 м.
5. Более дальние укрытия должны идти в `Укрытия для плана`, если юнит их видит.
6. Здания, камни, ящики/бочки, брёвна, заборы, посты и другие объекты должны попадать в укрытия, если они доступны по логике.
7. Расстояния в списках должны быть в метрах.

## Проверка физической высоты объектов

1. Поставить/найти объект: дом, дерево, камень, ящики/бочки, брёвна или забор.
2. Проверить Alt-линией видимости, что объект может быть причиной преграды.
3. Объект должен иметь физическую высоту `losHeightMeters`, а не быть бесконечной стеной.
4. С высоты низкие бочки/ящики внизу не должны автоматически закрывать весь обзор.
5. Дом и дерево должны закрывать видимость сильнее, чем низкие предметы.

## Проверка Alt-линии видимости

1. Выбрать солдата.
2. Зажать `Alt`.
3. Вести мышь от юнита в сторону карты.
4. Должна появиться линия:
   - зелёная часть — видит;
   - красная часть — дальше видимость прервана;
   - красный крест — место преграды.
5. Подпись должна быть чёткой HTML-плашкой, не пиксельным Pixi-текстом.
6. Подпись должна показывать:
   - `До курсора: ... м`;
   - `Видно: ... м`, если есть преграда;
   - причину преграды: плавный склон, лес, строение, камень, ящики/бочки и т.п.

## Проверка JSON сцены

1. В редакторе сделать небольшое изменение: высота, лес, предмет или зона.
2. Скачать JSON сцены.
3. Загрузить JSON сцены обратно.
4. Сцена должна восстановиться.
5. Экспорт должен сохранять:
   - `heightMap`;
   - `forestMap`;
   - `objects`;
   - `losHeightMeters` у объектов;
   - `units`;
   - `pressureZones`.

## Проверка отчёта производительности

1. В редакторе скачать отчёт производительности.
2. Отчёт должен содержать сведения о браузере, размере окна, renderer, scene, editor, FPS, frameMs/renderMs и худших кадрах.
3. Важный вывод: если renderMs низкий, а frameMs иногда высокий, проблема может быть не в самой отрисовке карты, а в браузере/компоновке/планировщике.

## GO / NO-GO

```text
GO, если:
- игра открывается через scripts/windows/run-preview.bat;
- карта выглядит как физическая карта, без россыпи +1 по умолчанию;
- игровой интерфейс вкладочный и не выглядит как длинная правая простыня;
- редактор отдельный, вкладочный, не ломает игровой режим;
- кисти высот/леса работают;
- JSON сцены скачивается и загружается обратно;
- слой Реальный рельеф включается и не тормозит от движения мыши;
- видимость считается по плавному рельефу, а не по линиям высот как по стенам;
- Alt-линия показывает метры, зелёный/красный участок и чёткую HTML-плашку;
- объекты учитываются как укрытия и преграды с физической высотой;
- ближние укрытия отделены от дальних укрытий для плана;
- лес 0/1/2 работает как слой карты и слой знаний юнита.

NO-GO, если:
- пользователю снова нужны Git/терминал-команды;
- интерфейс снова стал длинной простынёй справа;
- высоты снова отображаются как постоянные +1 на всей карте;
- слой Реальный рельеф заметно тормозит при движении мыши;
- линии высот снова работают как вертикальные стены;
- бочки/ящики/дома не учитываются в видимости или всегда работают как бесконечные стены;
- расстояния снова показаны в клетках вместо метров;
- редактор, JSON экспорт/загрузка или Alt-линия сломались.
```
