# Программа проверки — тонкие границы высот и загрузка JSON v2.4

Цель проверки: закрепить новую дефолтную сцену из JSON пользователя, сделать линии границ высот тонкими и однотонными, а также дать редактору возможность загрузить сцену из JSON-файла.

## Что открыть

Запускать из папки предпросмотра:

```text
scripts/windows/run-preview.bat
```

Или двойным кликом по корневому файлу:

```text
Run-Real-Wargame.bat
```

## Что должно быть на старте

1. Проект открывается в браузере.
2. На карте уже должен быть новый вариант пользователя из JSON:
   - холм с несколькими уровнями высоты;
   - редкий и густой лес;
   - один солдат возле холма;
   - предметы, добавленные в редакторе;
   - одна круглая зона редактора.
3. В обычном виде не должно быть постоянных надписей `+1`, `+2`, `-1` по клеткам.
4. Кнопка сверху должна показывать `Цифры высоты: выкл`.
5. В правом инспекторе при наведении на карту должны быть строки:
   - `Высота: ...`;
   - `Местность: ...`;
   - `Лес: ...`.

## Проверка качества границ высот

1. При обычном масштабе холм должен читаться как физическая карта, а не как набор квадратов.
2. Контурные линии высоты должны быть тонкими.
3. Контурные линии должны быть однотонными: без тёмных точек на более светлой линии.
4. Контурные линии не должны выглядеть как пунктир.
5. При зуме примерно `2x–3x` линия может немного сглаживаться, но не должна распадаться на точки.
6. Если точки всё ещё видны, это `NO-GO` для визуала и следующий шаг — перенос контуров в отдельный векторный Pixi Graphics слой.

## Проверка кисти высоты

1. Нажать кнопку `Редактор` внизу слева.
2. В правой панели должен появиться блок `Кисти карты: высота и лес`.
3. Нажать `Кисть: высота`.
4. В поле `Уровень высоты` выбрать `+2 холм`.
5. В поле `Размер кисти, клеток` поставить `5` или `7`.
6. Левой кнопкой мыши поводить по карте.
7. Должна появиться/расшириться область холма.
8. Выбрать `0 ровно / стереть высоту` и провести по части холма.
9. Эта часть должна вернуться к ровной земле.
10. Навести мышь на разные высоты: инспектор должен показывать правильную высоту.

## Проверка кисти леса

1. Нажать `Кисть: лес`.
2. В поле `Слой леса` выбрать `1 редкий лес`.
3. Провести по карте.
4. Должен появиться редкий лес.
5. Выбрать `2 густой лес`.
6. Провести рядом.
7. Густой лес должен быть темнее и плотнее редкого.
8. Выбрать `0 нет леса / стереть лес` и провести по лесу.
9. Лес должен стереться.

## Проверка загрузки JSON сцены

1. Нажать `Редактор`.
2. В правой панели найти блок `Сохранение / загрузка`.
3. Нажать `Загрузить JSON сцены`.
4. Выбрать ранее скачанный файл `real-wargame-scene-...json`.
5. Карта должна замениться содержимым файла.
6. После загрузки в сообщении редактора должно быть написано, что JSON сцены загружен.
7. После загрузки должны обновиться:
   - карта;
   - высоты;
   - лес;
   - предметы;
   - юниты;
   - зоны.

## Проверка сохранения

1. После правок нажать `Скачать JSON сцены`.
2. Браузер должен скачать файл `real-wargame-scene-...json`.
3. В скачанном JSON должны быть поля:
   - `map.heightMap`;
   - `map.forestMap`;
   - `map.objects`;
   - `units`;
   - `pressureZones`.
4. В `heightMap` должны быть уровни `-2..4`.
5. В `forestMap` должны быть значения:
   - `0` нет леса;
   - `1` редкий лес;
   - `2` густой лес.

## Проверка производительности

1. Покрутить зум 10–20 секунд.
2. Текст не должен заметно дёргаться.
3. Земля, рельеф и лес не должны мигать при обычном зуме.
4. Нажать `Скачать отчёт производительности`.
5. В отчёте производительности поле `renderer.antialias` должно быть `true`.
6. В отчёте производительности поле `renderer.maxFPS` должно быть `60`.

## GO / NO-GO

```text
GO, если:
- стартовая карта похожа на новый JSON пользователя;
- на карте один солдат;
- предметы и зона из JSON появились;
- границы высот тонкие, однотонные, без тёмных точек;
- можно рисовать и стирать высоты;
- можно рисовать и стирать редкий/густой лес;
- можно загрузить JSON сцены обратно в редактор;
- в инспекторе показаны высота, местность и лес;
- экспорт JSON содержит heightMap, forestMap, objects, units и pressureZones;
- зум не стал заметно хуже.

NO-GO, если:
- стартовая карта не похожа на новый JSON пользователя;
- слой высоты/леса стал мыльным или грубо пиксельным;
- контуры высот всё ещё выглядят как линия с тёмными точками;
- больше одного солдата;
- нельзя создать высоты или лес;
- нельзя загрузить JSON сцены;
- инспектор не показывает высоту/лес;
- экспорт сцены сломался;
- зум стал заметно хуже.
```
