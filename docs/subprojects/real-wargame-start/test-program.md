# Test program — Behavior Parameters v0.2

Цель проверки: убедиться, что preview-ветка считает параметры для будущего поведения ИИ, но пока не заставляет юнита застревать, ложиться или останавливаться.

## Что открыть

Запускать из preview-папки:

```text
scripts/windows/run-preview.bat
```

Или двойным кликом по корневому файлу:

```text
Run-Real-Wargame.bat
```

## Что проверить руками

1. Проект открывается в браузере.
2. Текст интерфейса по умолчанию русский.
3. Сетка визуально скрыта.
4. Сектор обзора визуально скрыт.
5. На карте видны полупрозрачные рыжие pressure-зоны.
6. Левым кликом выбирается юнит.
7. В правой панели появляется `Инспектор поведения`.
8. В инспекторе видны:
   - profile;
   - state;
   - posture;
   - action;
   - danger;
   - stress;
   - reason;
   - state reason;
   - posture reason;
   - last event;
   - thresholds.
9. Правым кликом дать юниту приказ пройти через pressure-зону.
10. Пока юнит реально входит в pressure-зону:
   - `danger` становится больше 0;
   - `stress` растёт;
   - юнит продолжает идти и не застревает;
   - posture не должна насильно менять движение;
   - state не должен останавливать движение.
11. Когда юнит выходит из pressure-зоны:
   - `danger` падает до 0;
   - `stress` постепенно снижается.
12. Групповое выделение рамкой продолжает работать.

## Ожидаемый результат

```text
Это ещё не поведение ИИ.
Это только слой параметров:
- pressure-zone считает danger/stress;
- inspector показывает параметры;
- юнит не застревает в зоне;
- движение не блокируется stress-ом;
- настоящее поведение будем задавать позже.
```

## Что не входит в этот пакет

```text
- автоматическое приседание/ложиться;
- остановка от stress;
- поиск укрытий;
- scoring-таблица вариантов действий;
- память/знания юнита;
- полноценный план командира.
```
