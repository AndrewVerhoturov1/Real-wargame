# Программа проверки — чистая карта, кисти высот и леса v2.2

Цель проверки: карта должна стать чистым полигоном для редактора. На старте — пустая ровная земля и один солдат. Высоты и лес создаются через кисти в редакторе. Рельеф должен иметь чёткие читаемые области, а не квадраты и не грязные мазки.

## Что открыть

Запускать из папки предпросмотра:

```text
scripts/windows/run-preview.bat
```

Или двойным кликом по корневому файлу:

```text
Run-Real-Wargame.bat
```

## Что должно быть на старте

1. Проект открывается в браузере.
2. На карте должна быть пустая ровная земля.
3. На карте должен быть только 1 солдат.
4. На карте не должно быть старых зон.
5. На карте не должно быть старых предметов: камней, постов, деревьев, заборов, домов.
6. В обычном виде не должно быть надписей `+1`, `+2`, `-1` по клеткам.
7. Кнопка сверху должна показывать `Цифры высоты: выкл`.
8. В правом инспекторе при наведении на карту должны быть строки:
   - `Высота: 0 — ровно`;
   - `Местность: поле / открытая земля`;
   - `Лес: нет`.

## Проверка кисти высоты

1. Нажать кнопку `Редактор` внизу слева.
2. В правой панели должен появиться блок `Кисти карты: высота и лес`.
3. Нажать `Кисть: высота`.
4. В поле `Уровень высоты` выбрать `+2 холм`.
5. В поле `Размер кисти, клеток` поставить `5` или `7`.
6. Левой кнопкой мыши поводить по карте.
7. Должна появиться область холма.
8. Граница холма должна быть не квадратной таблицей, а более кривой/сглаженной.
9. Цвет холма должен быть тёплый, жёлто-коричневый.
10. Выбрать `+4 гребень / вершина`.
11. Нарисовать внутри холма.
12. Должна появиться более высокая светлая область.
13. Навести мышь на верхнюю область.
14. Инспектор должен показать примерно `Высота: +4 — гребень / вершина`.
15. Выбрать `-2 глубокая низина`.
16. Нарисовать в другом месте.
17. Низина должна быть холоднее/темнее, сине-зелёная.
18. Навести мышь на низину.
19. Инспектор должен показать примерно `Высота: -2 — глубокая низина`.
20. Выбрать `0 ровно / стереть высоту` и провести по части холма.
21. Эта часть должна вернуться к ровной земле.

## Проверка формы рельефа

1. Нарисовать рядом несколько мазков высоты `+2` и `+3`.
2. Они должны объединяться в читаемую область рельефа.
3. Не должно быть ощущения случайного камуфляжа.
4. Не должно быть ощущения, что каждая клетка — отдельный цветной квадрат.
5. Граница между высотами должна быть заметной.
6. Граница должна быть кривой/сглаженной, насколько это возможно на текущей клеточной карте.
7. Цвета должны быть единообразны:
   - низ — холодный/тёмный;
   - 0 — почти без слоя;
   - верх — тёплый/светлый.

## Проверка кисти леса

1. Нажать `Кисть: лес`.
2. В поле `Слой леса` выбрать `1 редкий лес`.
3. Провести по карте.
4. Должен появиться редкий лес.
5. Выбрать `2 густой лес`.
6. Провести рядом.
7. Густой лес должен быть темнее и плотнее редкого.
8. Навести мышь на редкий лес.
9. Инспектор должен показать `Лес: редкий лес`.
10. Навести мышь на густой лес.
11. Инспектор должен показать `Лес: густой лес`.
12. Выбрать `0 нет леса / стереть лес` и провести по лесу.
13. Лес должен стереться.

## Проверка кнопок очистки

1. Нарисовать высоты и лес.
2. Нажать `Очистить высоты`.
3. Все высоты должны стать `0`, лес должен остаться.
4. Нажать `Очистить лес`.
5. Лес должен исчезнуть, высоты не должны измениться.

## Проверка цифр высоты

1. Нарисовать холм.
2. Нажать `Цифры высоты: выкл`.
3. Кнопка должна стать `Цифры высоты: вкл`.
4. На карте должны появиться отладочные цифры высоты.
5. Нажать кнопку ещё раз.
6. Цифры высоты должны исчезнуть, цветовой рельеф должен остаться.

## Проверка обычного редактора

1. В редакторе нажать `Вернуться к выбору` в блоке кистей.
2. В обычном блоке редактора выбрать `Создать предмет`.
3. Кликнуть по карте: предмет должен создаться.
4. Выбрать `Выбрать / тянуть`.
5. Потянуть предмет: он должен двигаться.
6. Удалить предмет.
7. Солдат должен оставаться на карте, если его не удаляли специально.

## Проверка сохранения

1. После рисования высот и леса нажать `Скачать JSON сцены`.
2. Браузер должен скачать файл `real-wargame-scene-...json`.
3. В скачанном JSON должны быть поля:
   - `map.heightMap`;
   - `map.forestMap`.
4. В `heightMap` должны быть нарисованные уровни `-2..4`.
5. В `forestMap` должны быть значения:
   - `0` нет леса;
   - `1` редкий лес;
   - `2` густой лес.

## Проверка производительности

1. Покрутить зум 10–20 секунд.
2. Текст не должен заметно дёргаться.
3. Земля, рельеф и лес не должны мигать при обычном зуме.
4. Нажать `Скачать отчёт производительности`.
5. В отчёте производительности поле `renderer.antialias` должно быть `true`.
6. В отчёте производительности поле `renderer.maxFPS` должно быть `60`.
7. В отчёте производительности поле `renderer.mapRender` должно говорить про `physical-map elevation bands` и `forest texture layer`.

## GO / NO-GO

```text
GO, если:
- стартовая карта чистая;
- на карте только 1 солдат;
- старые зоны и предметы исчезли;
- можно рисовать высоты кистью;
- можно стирать высоты кистью 0;
- можно рисовать редкий и густой лес;
- можно стирать лес кистью 0;
- в инспекторе показаны высота, местность и лес;
- рельеф читается лучше: низ/верх понятны;
- границы высот не выглядят как голые квадраты;
- экспорт JSON содержит heightMap и forestMap;
- зум не стал заметно хуже.

NO-GO, если:
- карта всё ещё захламлена;
- больше одного солдата;
- нельзя создать высоты;
- нельзя создать лес;
- рельеф опять выглядит как грязная мазня;
- высоты выглядят только квадратами без кривой границы;
- инспектор не показывает высоту/лес;
- экспорт сцены сломался;
- зум стал заметно хуже.
```

## Что не входит в этот пакет

```text
- автоматическая запись JSON прямо в файлы проекта;
- загрузка скачанного JSON обратно через интерфейс;
- влияние высоты на обзор, стрельбу и укрытия;
- влияние леса на видимость, скорость и скрытность;
- полноценный поиск пути.
```
