# Программа проверки — мягкий зум и отчёт производительности v1.7

Цель проверки: сделать стабильный зум приятнее без возврата к дрожащей анимации, а также добавить отладочный JSON-отчёт, который можно прислать для разбора тормозов. Зум теперь учитывает величину прокрутки колеса и делает более мягкий шаг; отчёт собирает последние кадры, FPS, время отрисовки, зум, число предметов, юнитов, зон и сведения о браузере.

## Что открыть

Запускать из папки предпросмотра:

```text
scripts/windows/run-preview.bat
```

Или двойным кликом по корневому файлу:

```text
Run-Real-Wargame.bat
```

## Что проверить руками

1. Проект открывается в браузере.
2. Текст интерфейса по умолчанию русский.
3. Сетка видна по умолчанию.
4. В левом верхнем углу карты видна неподвижная подпись масштаба: `1 клетка = 10 м`.
5. Покрутить колесо мыши для зума: масштаб должен меняться мягче, чем в прошлом ступенчатом варианте.
6. Зум не должен возвращаться к длинной плавной анимации через промежуточные кадры.
7. Текст не должен снова начать заметно дёргаться.
8. Подпись масштаба не должна увеличиваться/уменьшаться вместе с картой.
9. Правый интерфейс не должен заметно мерцать или давать эффект “тумана”.
10. Земля и сетка не должны мигать или пересоздаваться визуально.
11. Рыжие зоны не должны мигать или пересоздаваться визуально, если сами зоны не редактируются.
12. Внизу слева нажать плавающую кнопку `Редактор`.
13. После включения редактора игровые разделы должны скрыться.
14. Справа должен остаться отдельный раздел `Инспектор редактора`.
15. В режиме редактора подписи предметов и юнитов на карте должны быть скрыты, чтобы редактор меньше тормозил.
16. В инспекторе редактора должен быть блок `Сохранение`.
17. В блоке `Сохранение` должна быть кнопка `Скачать JSON сцены`.
18. В инспекторе редактора должен быть блок `Отладка производительности`.
19. В блоке `Отладка производительности` должна быть кнопка `Скачать отчёт производительности`.
20. Для проверки отчёта: покрутить зум 10–20 секунд, затем нажать `Скачать отчёт производительности`.
21. Браузер должен скачать файл с именем примерно `real-wargame-performance-...json`.
22. Внутри отчёта должны быть блоки:
    - `browser`;
    - `viewport`;
    - `renderer`;
    - `scene`;
    - `editor`;
    - `summary`;
    - `samples`.
23. В `summary` должны быть значения по FPS, времени кадра и времени отрисовки.
24. В `samples` должны быть последние замеры кадров.
25. Выбрать инструмент `Выбрать / тянуть`.
26. Кликнуть по предмету на карте: предмет должен подсветиться рамкой.
27. Зажать левую кнопку мыши в центре/на теле предмета и потянуть: предмет должен перемещаться, а не растягиваться.
28. Во время перетаскивания предмета земля и сетка не должны мигать или пересоздаваться визуально.
29. Выбрать инструмент `Создать зону`.
30. Кликнуть по карте: новая зона должна появиться в точке клика и стать выбранной.
31. Выбрать инструмент `Выбрать / тянуть`.
32. Кликнуть по зоне: зона должна подсветиться рамкой/ручками.
33. Зажать левую кнопку мыши внутри зоны и потянуть: зона должна перемещаться.
34. Нажать `Скачать JSON сцены`.
35. Браузер должен скачать файл с именем примерно `real-wargame-scene-...json`.

## Ожидаемый результат

```text
Зум:
- без длинной анимации через requestAnimationFrame;
- шаг зума мягче, чем раньше;
- величина прокрутки колеса влияет на силу изменения масштаба;
- обычная мышь должна давать более маленькие шаги;
- точное колесо/тачпад должны ощущаться мягче.

Отчёт производительности:
- можно скачать JSON-отчёт;
- в отчёте есть FPS, время кадра и время отрисовки;
- в отчёте есть браузер, размер окна, devicePixelRatio;
- в отчёте есть число предметов, юнитов и зон;
- в отчёте есть худшие кадры и худшие отрисовки.

PixiJS:
- канвас PixiJS создаётся без antialias;
- фон канваса непрозрачный;
- зум может выглядеть чуть жёстче, но должен быть легче.

Сохранение:
- редактор может скачать текущую сцену в JSON;
- JSON содержит карту, предметы, юниты и зоны;
- это ещё не автоматическая запись в файлы проекта.
```

## Что не входит в этот пакет

```text
- автоматический анализ отчёта внутри игры;
- перевод поля и предметов в Sprite / RenderTexture;
- автоматическая запись JSON прямо в файлы проекта;
- загрузка скачанного JSON обратно через интерфейс;
- перенос симуляции в отдельный рабочий поток;
- физическая коллизия с деревьями, домами, зонами и предметами;
- полноценный поиск пути.
```
