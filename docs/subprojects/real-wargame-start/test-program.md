# Программа проверки — пачки земли и стабильный зум v1.9

Цель проверки: исправить настоящую причину тяжёлого зума в карте. Отчёт v1.8 показал, что ограничение 45 FPS не помогло и даже дало частые кадры около 33–34 мс. Поэтому постоянный FPS возвращён к 60, а земля карты теперь рисуется пачками: не 2560 отдельных `Graphics` по одной клетке, а несколько крупных `Graphics` по типам местности.

## Что открыть

Запускать из папки предпросмотра:

```text
scripts/windows/run-preview.bat
```

Или двойным кликом по корневому файлу:

```text
Run-Real-Wargame.bat
```

## Что проверить руками

1. Проект открывается в браузере.
2. Текст интерфейса по умолчанию русский.
3. Сетка видна по умолчанию.
4. Кнопка сверху должна показывать `Обзор: выкл`.
5. В новом отчёте производительности поле `renderer.viewCones` должно быть `false`, если ты сам не включал обзор.
6. В новом отчёте производительности поле `renderer.maxFPS` должно быть `60`.
7. В новом отчёте производительности поле `renderer.mapRender` должно говорить про `batched Pixi Graphics`.
8. В левом верхнем углу карты видна неподвижная подпись масштаба: `1 клетка = 10 м`.
9. Покрутить колесо мыши для зума 10–20 секунд.
10. Зум должен меньше подвисать, чем в отчёте `v1.8`.
11. Зум не должен возвращаться к длинной плавной анимации через промежуточные кадры.
12. Текст не должен снова начать заметно дёргаться.
13. Подпись масштаба не должна увеличиваться/уменьшаться вместе с картой.
14. Правый интерфейс не должен заметно мерцать или давать эффект “тумана”.
15. Земля и сетка не должны мигать или пересоздаваться визуально.
16. Рыжие зоны не должны мигать или пересоздаваться визуально, если сами зоны не редактируются.
17. Внизу слева нажать плавающую кнопку `Редактор`.
18. После включения редактора игровые разделы должны скрыться.
19. Справа должен остаться отдельный раздел `Инспектор редактора`.
20. В режиме редактора подписи предметов и юнитов на карте должны быть скрыты, чтобы редактор меньше тормозил.
21. В инспекторе редактора должен быть блок `Сохранение`.
22. В блоке `Сохранение` должна быть кнопка `Скачать JSON сцены`.
23. В инспекторе редактора должен быть блок `Отладка производительности`.
24. В блоке `Отладка производительности` должна быть кнопка `Скачать отчёт производительности`.
25. Для проверки отчёта: покрутить зум 10–20 секунд, затем нажать `Скачать отчёт производительности`.
26. Браузер должен скачать файл с именем примерно `real-wargame-performance-...json`.
27. Внутри отчёта должны быть блоки:
    - `browser`;
    - `viewport`;
    - `renderer`;
    - `scene`;
    - `editor`;
    - `summary`;
    - `samples`.
28. В `summary.frameMs.p95` желательно должно стать ближе к 16–18 мс, а не около 34 мс.
29. В `summary.renderMs.avg` отрисовка должна оставаться маленькой.
30. Выбрать инструмент `Выбрать / тянуть`.
31. Кликнуть по предмету на карте: предмет должен подсветиться рамкой.
32. Зажать левую кнопку мыши в центре/на теле предмета и потянуть: предмет должен перемещаться, а не растягиваться.
33. Во время перетаскивания предмета земля и сетка не должны мигать или пересоздаваться визуально.
34. Выбрать инструмент `Создать зону`.
35. Кликнуть по карте: новая зона должна появиться в точке клика и стать выбранной.
36. Выбрать инструмент `Выбрать / тянуть`.
37. Кликнуть по зоне: зона должна подсветиться рамкой/ручками.
38. Зажать левую кнопку мыши внутри зоны и потянуть: зона должна перемещаться.
39. Нажать `Скачать JSON сцены`.
40. Браузер должен скачать файл с именем примерно `real-wargame-scene-...json`.

## Ожидаемый результат

```text
По отчёту v1.8:
- 45 FPS не помогли;
- появились частые кадры 33–34 мс;
- значит, ограничение FPS было плохим решением для ощущения зума.

Что изменено:
- постоянный тик сцены возвращён к 60 FPS;
- земля карты рисуется пачками по типам местности;
- вместо 2560 отдельных Graphics-клеток должно быть примерно до 6 больших Graphics-слоёв земли;
- сетка всё ещё отдельный слой;
- предметы и зоны пока остаются отдельными векторными объектами.

Зум:
- без длинной анимации через requestAnimationFrame;
- шаг зума мягкий;
- величина прокрутки колеса влияет на силу изменения масштаба;
- текст не должен заметно дёргаться.

Отчёт производительности:
- можно скачать JSON-отчёт;
- в отчёте есть FPS, время кадра и время отрисовки;
- в отчёте есть браузер, размер окна, devicePixelRatio;
- в отчёте есть число предметов, юнитов и зон;
- в отчёте есть худшие кадры и худшие отрисовки.

PixiJS:
- канвас PixiJS создаётся без antialias;
- фон канваса непрозрачный;
- земля больше не является тысячами отдельных DisplayObject.

Сохранение:
- редактор может скачать текущую сцену в JSON;
- JSON содержит карту, предметы, юниты и зоны;
- это ещё не автоматическая запись в файлы проекта.
```

## Что не входит в этот пакет

```text
- перевод земли в RenderTexture / Sprite;
- перевод предметов в Sprite;
- полностью событийная отрисовка без постоянного тика;
- автоматическая запись JSON прямо в файлы проекта;
- загрузка скачанного JSON обратно через интерфейс;
- перенос симуляции в отдельный рабочий поток;
- физическая коллизия с деревьями, домами, зонами и предметами;
- полноценный поиск пути.
```
