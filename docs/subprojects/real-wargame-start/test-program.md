# Test program — Behavior Tick v0.2

Цель проверки: убедиться, что preview-ветка показывает динамическую реакцию юнитов на pressure-зоны и готовый инспектор для первых тестов ИИ.

## Что открыть

Запускать из preview-папки:

```text
scripts/windows/run-preview.bat
```

Или двойным кликом по корневому файлу:

```text
Run-Real-Wargame.bat
```

## Что проверить руками

1. Проект открывается в браузере.
2. На карте видны полупрозрачные рыжие pressure-зоны.
3. Левым кликом выбирается юнит.
4. В правой панели появляется `Behavior inspector` / `Инспектор поведения`.
5. В инспекторе видны:
   - profile;
   - state;
   - posture;
   - action;
   - danger;
   - stress;
   - reason;
   - state reason;
   - posture reason;
   - last event;
   - thresholds.
6. Правым кликом дать юниту приказ пройти через pressure-зону.
7. Пока юнит реально входит в pressure-зону:
   - `danger` становится больше 0;
   - `stress` растёт;
   - posture меняется на `crouched` или `prone`;
   - state меняется на `moving`, `taking_cover` или `stressed`;
   - скорость движения визуально снижается при низком положении.
8. Когда юнит выходит из pressure-зоны:
   - `danger` падает до 0;
   - `stress` постепенно снижается;
   - posture возвращается к `standing`, когда давление стало приемлемым.
9. При очень высоком stress юнит должен перейти в `stressed`, стать `prone` и временно остановиться.
10. Групповое выделение рамкой продолжает работать.
11. Переключатели сетки и сектора обзора продолжают работать.

## Ожидаемый результат

```text
Это ещё не полноценный ИИ-командир.
Это первый динамический слой поведения:
- у юнитов есть behavior profile;
- у юнитов есть state/posture/stress/danger;
- stress растёт и падает во времени;
- posture и state меняются во время движения;
- inspector объясняет, почему юнит изменил поведение.
```

## Что не входит в этот пакет

```text
- поиск укрытий;
- сложный редактор поведения;
- scoring-таблица вариантов действий;
- память/знания юнита;
- полноценный план командира.
```

Эти пункты должны идти следующими отдельными r-init задачами.
