# Программа проверки — читаемый рельеф высот v2.1.1

Цель проверки: исправить первый вариант рельефа. На карте больше не должно быть ощущения камуфляжных мазков и непонятных зелёно-жёлтых пятен. Высоты должны читаться как единая шкала: низ — холоднее/темнее, верх — теплее/светлее.

## Что открыть

Запускать из папки предпросмотра:

```text
scripts/windows/run-preview.bat
```

Или двойным кликом по корневому файлу:

```text
Run-Real-Wargame.bat
```

## Что проверить руками

1. Проект открывается в браузере.
2. Текст интерфейса по умолчанию русский.
3. Кнопка сверху должна показывать `Цифры высоты: выкл`.
4. В обычном виде на карте не должно быть постоянных надписей `+1`, `+2`, `-1` по клеткам.
5. Рельеф не должен выглядеть как камуфляж или случайные мазки.
6. Цвета высот должны быть единообразными:
   - `-2` — глубокая низина, тёмный холодный сине-зелёный;
   - `-1` — низина, приглушённый холодный тон;
   - `0` — ровно, почти без цветового слоя;
   - `+1` — лёгкий подъём, слабый тёплый тон;
   - `+2` — холм, заметнее тёплый тон;
   - `+3` — высокая местность, ещё светлее/теплее;
   - `+4` — гребень/вершина, самый светлый тёплый тон.
7. Один и тот же цвет не должен означать то верх, то низ.
8. Большие прямоугольники леса/болота/пересечённой земли не должны сами по себе считаться высотой.
9. Возле наблюдательного поста должна быть заметная высокая местность.
10. В болотистой зоне должны быть небольшие низкие участки.
11. Навести мышь на разные клетки карты.
12. В правом инспекторе должна появляться строка `Высота: ...`.
13. Для клетки высоты `+4` инспектор должен писать примерно `Высота: +4 — гребень / вершина`.
14. Для ровной клетки инспектор должен писать примерно `Высота: 0 — ровно`.
15. В инспекторе также должны быть строки `Местность: ...` и `Лес: ...`.
16. Нажать `Цифры высоты: выкл`.
17. Кнопка должна стать `Цифры высоты: вкл`.
18. После включения должны появиться отладочные цифры высоты.
19. Нажать кнопку ещё раз.
20. Цифры высоты должны исчезнуть, а читаемый цветовой рельеф остаться.
21. Круги зон, линии, предметы и рамки должны оставаться мягкими: сглаживание не должно быть выключено.
22. Покрутить колесо мыши для зума 10–20 секунд.
23. Текст не должен снова начать заметно дёргаться.
24. Земля, рельеф и сетка не должны мигать при обычном зуме.
25. Внизу слева нажать плавающую кнопку `Редактор`.
26. После включения редактора игровые разделы должны скрыться.
27. Справа должен остаться отдельный раздел `Инспектор редактора`.
28. В режиме редактора подписи предметов и юнитов на карте должны быть скрыты.
29. Выбрать инструмент `Выбрать / тянуть`.
30. Кликнуть по предмету на карте: предмет должен подсветиться рамкой.
31. Зажать левую кнопку мыши в центре/на теле предмета и потянуть: предмет должен перемещаться, а не растягиваться.
32. Нажать `Скачать JSON сцены`.
33. Браузер должен скачать файл с именем примерно `real-wargame-scene-...json`.
34. В скачанном JSON должны быть поля:
    - `map.heightMap`;
    - `map.forestMap`.
35. Нажать `Скачать отчёт производительности`.
36. Браузер должен скачать файл с именем примерно `real-wargame-performance-...json`.
37. В отчёте производительности поле `renderer.antialias` должно быть `true`.
38. В отчёте производительности поле `renderer.maxFPS` должно быть `60`.
39. В отчёте производительности поле `renderer.mapRender` должно говорить про `readable smoothed elevation texture`.

## Ожидаемый результат

```text
Карта:
- высоты поддерживают уровни -2, -1, 0, 1, 2, 3, 4;
- обычный вид карты больше не засыпан цифрами +1/+2/-1;
- цифры высоты включаются только отдельной отладочной кнопкой;
- рельеф строится как единая сглаженная высотная карта, а не как случайные мазки;
- в инспекторе видно, какая высота под мышью;
- данные остаются клеточными, чтобы потом использовать их для обзора, укрытий и движения солдат.

Рендер:
- базовая земля остаётся пачками по типам местности;
- рельеф собирается в отдельный canvas/texture-слой;
- цвет высоты выбирается из одной постоянной шкалы;
- сетка, предметы, зоны, юниты и приказы остаются отдельными слоями;
- сглаживание PixiJS остаётся включённым.

Сохранение:
- экспорт сцены остаётся scene-export-v2;
- JSON содержит heightMap;
- JSON содержит forestMap как заготовку под будущие слои леса: 0 нет леса, 1 редкий лес, 2 густой лес.
```

## GO / NO-GO

```text
GO, если:
- сразу понятнее, где низина, где ровно, где холм и гребень;
- нет ощущения камуфляжа/грязных мазков;
- один цветовой смысл не скачет от клетки к клетке;
- инспектор показывает высоту клетки под мышью;
- цифры высоты выключены по умолчанию и включаются только кнопкой;
- редактор предметов и зон работает;
- экспорт JSON сцены работает;
- зум не стал заметно хуже.

NO-GO, если:
- рельеф всё ещё выглядит как случайная мазня;
- зелёные/жёлтые пятна не дают понять, где верх, а где низ;
- большие прямоугольники опять воспринимаются как высота;
- инспектор не показывает высоту;
- кнопка цифр высоты не работает;
- экспорт сцены сломался;
- зум стал заметно хуже.
```

## Что не входит в этот пакет

```text
- кисти высоты в редакторе;
- кисти редкого/густого леса;
- автоматическая запись JSON прямо в файлы проекта;
- загрузка скачанного JSON обратно через интерфейс;
- влияние высоты на обзор, стрельбу и укрытия;
- влияние леса на видимость, скорость и скрытность;
- полноценный поиск пути.
```
