# Программа проверки — плавный рельеф, укрытия и линия видимости v0.6

Цель проверки: линии высот остаются только визуальной картой, а расчёт видимости использует плавный реальный рельеф. В интерфейсе есть слой `Реальный рельеф`, подпись линии видимости выводится чётким HTML-текстом, объекты считаются укрытиями, а лес в знаниях юнита показывается: ближний — квадратами, дальний — кругами.

## Что открыть

Запускать из папки предпросмотра:

```text
scripts/windows/run-preview.bat
```

## Проверка реального рельефа

1. Открыть preview.
2. В верхнем меню нажать `Реальный рельеф: выкл`.
3. Кнопка должна стать `Реальный рельеф: вкл`.
4. На карте должен появиться полупрозрачный слой плавной фактической высоты.
5. Этот слой показывает не линии контуров, а реальный сглаженный склон для расчёта видимости.
6. Линии высот на карте остаются только визуальной подсказкой, не стеной.

## Проверка игрового режима

1. Выбрать солдата.
2. Открыть правую вкладку `Слои`.
3. В списках расстояния должны быть в метрах, не в клетках.
4. Должны быть отдельные разделы:
   - `Ближние укрытия`;
   - `Укрытия для плана`;
   - `Опасность`.
5. На карте должен появиться оверлей знаний:
   - зелёный круг — область обзора юнита;
   - жёлтые квадраты — ближние укрытия;
   - жёлтые круги — дальние известные укрытия / ориентиры;
   - красные метки — известная опасность.

## Проверка укрытий

1. Ближние укрытия должны быть только рядом с юнитом — примерно до 35 м.
2. Более дальние укрытия должны идти в `Укрытия для плана`, если юнит видит их по линии видимости.
3. Здания, камни, ящики/бочки, брёвна, заборы, посты и другие объекты должны попадать в укрытия, если они доступны по логике.
4. Лес рядом должен показываться подробнее — отдельными ближними клетками.
5. Дальний лес должен показываться компактно — одним ориентиром на массив, а не сотней меток.

## Проверка опасности

1. Опасность не должна считаться только по расстоянию.
2. Опасность должна появляться как видимая, если есть линия видимости.
3. Если юнит внутри опасной зоны, она может быть известна как давление зоны.

## Проверка Alt-линии видимости

1. Выбрать солдата.
2. Зажать `Alt`.
3. Вести мышь от юнита в сторону карты.
4. Должна появиться линия:
   - зелёная часть — видит;
   - красная часть — дальше видимость прервана;
   - красный крест — место преграды.
5. Подпись должна быть чёткой, не пиксельной.
6. Подпись должна показывать:
   - `До курсора: ... м`;
   - `Видно: ... м`, если есть преграда;
   - причину преграды: плавный склон, лес, строение, камень и т.п.

## Проверка редактора

1. Нажать `Режим: игра` сверху.
2. Должен открыться редактор.
3. Редактор должен остаться во вкладках, а не длинной простынёй.
4. Кисти высот/леса должны быть во вкладке `Карта`.
5. JSON и отчёт производительности должны быть во вкладке `Сцена`.

## GO / NO-GO

```text
GO, если:
- расстояния показываются в метрах;
- есть верхняя кнопка Реальный рельеф;
- видимость считается по плавному рельефу, а не по линиям высот как по стенам;
- подпись Alt-линии чёткая HTML-плашка;
- ближние укрытия отделены от дальних укрытий для плана;
- здания, бочки/ящики и другие объекты попадают в укрытия;
- ближний лес показан квадратами, дальний лес — кругами-ориентирами;
- редактор не вернулся к длинной простыне;
- игра открывается и скриншот-проверка проходит.

NO-GO, если:
- где-то в списках укрытий снова показаны клетки вместо метров;
- линия высоты всё ещё ведёт себя как вертикальная стена;
- слой реального рельефа не включается;
- подпись Alt-линии снова мыльная/пиксельная;
- дальнее укрытие считается текущим укрытием;
- объекты не попадают в список укрытий;
- Alt-линия не появляется;
- редактор или экспорт JSON сломались.
```
