# Программа проверки — стабильный ступенчатый зум v1.5

Цель проверки: сравнить новый режим зума без анимации. Теперь колесо мыши делает один небольшой стабильный шаг масштаба сразу, без промежуточных кадров. Это должно убрать ощущение, что весь текст “перещёлкивает позиции” во время плавного зума.

## Что открыть

Запускать из папки предпросмотра:

```text
scripts/windows/run-preview.bat
```

Или двойным кликом по корневому файлу:

```text
Run-Real-Wargame.bat
```

## Что проверить руками

1. Проект открывается в браузере.
2. Текст интерфейса по умолчанию русский.
3. Сетка видна по умолчанию.
4. В левом верхнем углу карты видна неподвижная подпись масштаба: `1 клетка = 10 м`.
5. Покрутить колесо мыши для зума: масштаб должен меняться маленькими шагами, без плавной анимации между ними.
6. Во время зума не должно быть длинной серии промежуточных кадров, где текст постепенно едет и перещёлкивает позиции.
7. Подпись масштаба не должна увеличиваться/уменьшаться вместе с картой.
8. Подпись масштаба не должна дрожать.
9. Правый интерфейс не должен заметно мерцать или давать эффект “тумана”.
10. Текстовые подписи на карте должны стоять стабильнее, потому что их координаты округляются до целых пикселей.
11. Земля и сетка не должны мигать или пересоздаваться визуально.
12. Рыжие зоны не должны мигать или пересоздаваться визуально, если сами зоны не редактируются.
13. Внизу слева нажать плавающую кнопку `Редактор`.
14. После включения редактора игровые разделы должны скрыться.
15. Справа должен остаться отдельный раздел `Инспектор редактора`.
16. В режиме редактора подписи предметов и юнитов на карте должны быть скрыты, чтобы редактор меньше тормозил.
17. В инспекторе редактора должен быть блок `Сохранение`.
18. В блоке `Сохранение` должна быть кнопка `Скачать JSON сцены`.
19. Выбрать инструмент `Выбрать / тянуть`.
20. Кликнуть по предмету на карте: предмет должен подсветиться рамкой.
21. Зажать левую кнопку мыши в центре/на теле предмета и потянуть: предмет должен перемещаться, а не растягиваться.
22. Во время перетаскивания предмета земля и сетка не должны мигать или пересоздаваться визуально.
23. Выбрать инструмент `Создать зону`.
24. Кликнуть по карте: новая зона должна появиться в точке клика и стать выбранной.
25. Выбрать инструмент `Выбрать / тянуть`.
26. Кликнуть по зоне: зона должна подсветиться рамкой/ручками.
27. Зажать левую кнопку мыши внутри зоны и потянуть: зона должна перемещаться.
28. Нажать `Скачать JSON сцены`.
29. Браузер должен скачать файл с именем примерно `real-wargame-scene-...json`.

## Ожидаемый результат

```text
Зум:
- нет плавной анимации через requestAnimationFrame;
- один поворот колеса = один небольшой шаг масштаба;
- меньше шансов увидеть, как текст едет по промежуточным позициям;
- зум может быть менее “красивым”, зато должен быть стабильнее.

Текст:
- подпись масштаба не масштабируется вместе с картой;
- подпись масштаба не дрожит при зуме;
- правый интерфейс без размытого фона;
- HTML-подписи карты ставятся в целые пиксели;
- строка масштаба в инспекторе обновляется реже и грубее: 1 знак после запятой.

Карта:
- bitmap-кэш статичной карты отключён, чтобы не было эффекта мутной картинки через туман;
- земля и сетка всё ещё не пересобираются при каждом движении предмета;
- зоны параметров вынесены в отдельный слой;
- подсветка клетки мыши и рамка выделения не заставляют пересобирать зоны.

Сохранение:
- редактор может скачать текущую сцену в JSON;
- JSON содержит карту, предметы, юниты и зоны;
- это ещё не автоматическая запись в файлы проекта.
```

## Что не входит в этот пакет

```text
- автоматическая запись JSON прямо в файлы проекта;
- загрузка скачанного JSON обратно через интерфейс;
- перенос симуляции в отдельный рабочий поток;
- физическая коллизия с деревьями, домами, зонами и предметами;
- полноценный поиск пути.
```
