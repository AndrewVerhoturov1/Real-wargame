# Восприятие и внимание бойца v1

## Назначение

Система отделяет физическую линию обзора от фактического обнаружения. Свободный луч до цели больше не означает, что боец мгновенно знает её положение.

Цепочка расчёта:

```text
физическая видимость
→ визуальный или звуковой сигнал
→ текущее поле внимания
→ накопление признаков
→ субъективный контакт
→ стареющая память
```

Версия v1 рассчитывает восприятие только для выбранного бойца. Это осознанное ограничение текущего вертикального среза, а не модель восприятия всей армии.

## Гибрид A+B

Игрок видит три понятные зоны:

1. **Фокус** — узкое направление, которое боец изучает особенно внимательно.
2. **Прямое внимание** — широкий передний сектор.
3. **Косвенное внимание** — слабое восприятие событий за пределами переднего сектора.

Внутри симуляции сила внимания меняется плавно. На границе нарисованного сектора нет резкого переключения «видит / не видит».

## Режимы внимания

| Режим | Поведение |
|---|---|
| Марш | Широкий передний контроль и полезное круговое косвенное внимание. Лучше замечает движение и выстрелы, чем неподвижные детали. |
| Наблюдение | Спокойно сканирует широкий передний сектор. |
| Поиск цели | Проводит узким фокусом по заданному сектору. |
| Стрельба | Удерживает внимание на текущей цели, значительно хуже контролируя фланги и тыл. |

Постоянные углы, веса и интервалы задаются в игровом редакторе бойца. Ноды ИИ только выбирают режим или сектор поиска.

## Настройки профиля

Для каждого режима доступны:

- угол фокуса;
- угол прямого внимания;
- сила фокуса;
- сила прямого внимания;
- сила косвенного внимания;
- скорость осмотра;
- интервалы проверки фокуса, прямого сектора и периферии;
- частота проверки тыла;
- стандартная ширина сектора поиска.

Старые сцены без блока `attention` получают безопасные значения по умолчанию.

## Заметность цели

Скорость накопления признаков зависит от именованных факторов:

- поза цели;
- движение;
- действие, особенно стрельба;
- размер;
- маскировка;
- расстояние;
- поперечное движение;
- направление внимания;
- зрение, внимание и состояние наблюдателя;
- частичное закрытие лесом;
- усталость, растерянность и подавление.

Каждый фактор сохраняет русское объяснение для панели диагностики.

## Растительность и препятствия

Рельеф, стены и достаточно высокие объекты полностью блокируют зрительный сигнал.

Лес действует постепенно. Луч получает значение `visualTransmission` от 1 до 0. Короткая полоса растительности ухудшает обнаружение, а достаточно длинная или густая становится практически непрозрачной.

Укрытие и маскировка остаются разными понятиями.

## Стадии контакта

```text
признак → подозрение → контакт → опознано → подтверждено
```

Начальные пороги доказательств:

| Стадия | Значение |
|---|---:|
| Признак | 25 |
| Подозрение | 50 |
| Контакт | 80 |
| Опознано | 120 |
| Подтверждено | 150 |

После потери цели доказательства и уверенность уменьшаются, а область возможного положения расширяется.

## Слух

Временные звуковые события поддерживают:

- одиночный выстрел;
- автоматический огонь;
- взрыв;
- движение.

Звук создаёт приблизительный контакт и не даёт точных координат. Контакт только по звуку не поднимается выше подозрения. При последующем визуальном обнаружении данные объединяются по источнику.

## Субъективные знания

Физическая опасность и знание об источнике разделены.

Боец может получать подавление и стресс от огня, но его Blackboard не получает точную позицию стрелка до накопления контакта, доклада или приблизительного вывода по обстрелу.

`sourceVisible` означает, что источник способен создавать визуальный сигнал. Это не автоматическое обнаружение.

`sourceKnown` создаёт переданный или сценарный контакт, но не текущую зрительную видимость.

## Blackboard

Добавлены значения:

- `attention_mode`;
- `attention_focus_direction`;
- `best_contact_stage`;
- `best_contact_confidence`;
- `best_contact_uncertainty` в метрах;
- `contact_visible_now`;
- `suspected_enemy_position`.

`current_target`, `known_enemy_position`, `enemyVisible` и `enemyKnown` теперь опираются на личные контакты бойца, а не напрямую на объективный центр зоны угрозы.

## Ноды ИИ

- **Выбрать режим внимания** (`SetAttentionMode`)
- **Задать сектор поиска** (`SetSearchSector`)
- **Вернуть автоматическое внимание** (`ClearAttentionOverride`)

Ноды не содержат коэффициенты профиля и не раскрывают неизвестных противников.

## Интерфейс

В симуляции доступна вкладка **Внимание**. Она показывает:

- текущий режим и источник режима;
- направление фокуса;
- углы и силу косвенного внимания;
- ход сканирования;
- лучший контакт, уверенность и неточность;
- текущую скорость накопления;
- русское объяснение факторов;
- список контактов в памяти.

Отдельный PixiJS-слой рисует фокус, прямой сектор, периферийный круг и контакты только для выбранного бойца.

## Производительность

- восприятие не считается от движения камеры или курсора;
- фокус, прямой сектор и периферия имеют разные интервалы проверки;
- угол проверяется до дорогого LOS;
- в v1 работает только выбранный боец;
- отрисовщик имеет собственный ключ кэша;
- нагрузочный smoke-тест использует 120 источников и 600 тиков без требования нестабильного времени выполнения.

## Ограничения v1

Пока не реализованы:

- восприятие каждого бойца всей армии;
- командная передача контактов;
- бинокли, оптика и приборы техники;
- подробная ночь и погода;
- отдельные вражеские юниты как полноценная боевая сторона;
- полная баллистика и поражение целей.

## Проверки

Основные команды:

```text
npm run perception:smoke
npm run perception-performance:smoke
npm run attention-ai-nodes:smoke
npm run build
npm run runtime:smoke
npm run workspace:smoke
npm run game-editor:smoke
npm run visibility-probe:smoke
npm run dictionary:smoke
```

Визуальная проверка должна использовать реальное приложение и свежие PNG для марша, поиска и стрельбы. Движение курсора отдельно проверяется на отсутствие перестройки оверлея.
