# Stateful AI Runtime v1

Дата: 2026-07-11  
Статус: реализовано и проверено  
Подпроект: `ai-single-unit-editor`

## Что изменилось

До этого `AiGraphRunner` выполнял граф как мгновенный расчёт: нода либо проходила, либо проваливалась за один вызов.

Теперь поверх него появился отдельный чистый исполнитель:

```text
src/core/ai/AiGraphRuntime.ts
```

Он умеет сохранять ход выполнения между тиками ИИ и возвращает состояния:

```text
success    — действие успешно завершено;
failure    — действие провалилось;
running    — контракт для активного длительного действия;
waiting    — нода ожидает время или событие;
cancelled  — действие корректно отменено.
```

В первой версии реальной длительной нодой является `Wait`, поэтому живой граф сейчас использует `waiting`. Статус `running` подготовлен для следующей длительной ноды движения.

## Новые ноды

### `SequenceWithMemory`

Русское название:

```text
Последовательность с памятью
```

Она выполняет дочерние шаги по порядку и запоминает, на каком шаге остановилась.

Пример:

```text
Пригнуться
→ Осмотреться 4 секунды
→ Двигаться к укрытию
```

На следующем тике граф не начинает цепочку заново, а продолжает активный шаг.

### `Wait`

Русское название:

```text
Ждать
```

Параметры:

```text
durationSeconds — длительность ожидания;
timeoutSeconds  — тайм-аут, 0 означает без тайм-аута.
```

Обычный пользователь настраивает оба значения в русской форме. Редактировать JSON не требуется.

## Жизненный цикл

Первый контракт жизненного цикла:

```text
start     — нода запущена;
update    — нода продолжает работу;
complete  — нода завершена;
cancel    — нода отменена.
```

Состояние выполнения хранит:

- идентификатор графа;
- идентификатор бойца;
- выбранную ветку;
- последовательность;
- номер активного шага;
- активную ноду;
- время начала;
- время последнего обновления;
- текущее состояние.

Это отдельное serializable-состояние. Оно не смешано с Blackboard и не содержит DOM, PixiJS или игровые объекты.

## Отмена

Добавлен явный контракт отмены.

При отмене активной ноды:

- записывается событие `cancel`;
- состояние выполнения очищается;
- старые effects не применяются повторно;
- редактор показывает причину отмены.

У `Wait` пока нет внешних ресурсов. Следующая длительная нода движения обязана при отмене очищать маршрут, цель и будущие резервы позиции.

## Совместимость

Сохранено:

- graph version `1`;
- localStorage graph v6;
- старый формат scene export/import;
- старые графы без длительных нод;
- текущий `UtilitySelector` и разложение очков;
- чистота `AiGraphRunner.ts`.

Старый мгновенный исполнитель не переписан. `AiGraphRuntime` использует его для выбора Utility-ветки и выполнения обычных мгновенных дочерних нод.

## Интеграция с игрой

`AiGameBridge.ts` теперь:

1. строит Blackboard выбранного бойца;
2. передаёт сохранённое execution state в `AiGraphRuntime`;
3. получает новый статус и состояние;
4. сохраняет состояние только при реальном игровом тике;
5. применяет effects текущего тика;
6. публикует живую диагностику в AI Node Editor.

Preview-only `Evaluate` не изменяет живое состояние бойца.

Как и раньше, автоматическое выполнение пока работает только для выбранного бойца.

## AI Node Editor

Добавлено отображение:

- жёлтый — `running`;
- синий — `waiting`;
- зелёный — завершено успешно;
- красный — провал;
- оранжевый — отменено.

Панель `След ИИ` показывает:

- состояние исполнения;
- активную ноду;
- прошедшее время;
- причину отмены;
- выбранную Utility-ветку и её очки.

Для `Wait` на первом экране инспектора находятся поля:

```text
Длительность, секунд
Тайм-аут, секунд
```

Значения сохраняются через обычную кнопку `Сохранить параметры`.

## Local AI Engine

`Evaluate once` не изображает, будто время прошло.

Для графа с длительными нодами local engine:

- проверяет мгновенную структуру;
- останавливается на границе длительного исполнения;
- сообщает, что живой прогресс `Выполняется / Ожидает` нужно смотреть через игру и `AiGraphRuntime`.

## Ограничения v1

Пока нет:

- настоящей длительной ноды движения со статусом `running`;
- реактивной перепроверки условий во время действия;
- событийной подписки на Blackboard;
- параллельных ветвей;
- владения каналами движения, оружия и позы;
- резервирования укрытий;
- сохранения execution state в scene JSON;
- выполнения графа для всей армии.

`SequenceWithMemory` v1 исполняет одну первую найденную состоянийную последовательность в выбранной ветке. Вложенные состояния и несколько независимых длительных цепочек отложены.

## Следующий рекомендуемый вертикальный срез

```text
MoveToBlackboardPosition
```

Нода должна:

```text
start
→ один раз назначить цель и маршрут;

update
→ возвращать running и проверять прибытие;

complete
→ завершиться при достижении позиции;

cancel
→ снять цель, маршрут и резерв позиции.
```

Первый игровой сценарий:

```text
Под огнём
→ найти укрытие
→ пригнуться
→ двигаться к укрытию несколько тиков
→ достигнуть укрытия
→ лечь
```

После этого следует добавить реактивную отмену при новом приказе, потере укрытия, блокировке пути или критическом изменении угрозы.

## Правила продолжения

- Работать только в `real-wargame-preview`.
- `main` не трогать без прямого GO пользователя.
- Не добавлять длительное действие без корректной очистки при `cancel`.
- Не запускать граф для всей армии до стабилизации одного бойца.
- Не считать наличие статуса `running` доказательством готового длительного движения: оно является следующим этапом.
- UI считать проверенным только после реального браузера и открытия свежего PNG.
