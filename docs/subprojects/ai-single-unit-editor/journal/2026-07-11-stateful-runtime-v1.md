# Journal — Stateful AI Runtime v1

Дата: 2026-07-11

## Исходная проблема

`AiGraphRunner` хорошо выбирал лучшее действие через Utility AI, но каждый вызов был мгновенным и не помнил активный шаг. Нельзя было честно выразить:

```text
начал действие
→ продолжаю действие
→ завершил или отменил действие
```

## Принятое решение

Сохранить существующий мгновенный `AiGraphRunner` и добавить поверх него чистый `AiGraphRuntime`.

Это позволило:

- не переписывать проверенный UtilitySelector;
- сохранить старые графы;
- отделить постоянный Blackboard от временного execution state;
- подготовить отмену до появления маршрутов и резервов;
- оставить core независимым от DOM и PixiJS.

## Реализовано

- статусы `success / failure / running / waiting / cancelled`;
- lifecycle `start / update / complete / cancel`;
- serializable execution state выбранного бойца;
- `SequenceWithMemory`;
- `Wait` с длительностью и тайм-аутом;
- безопасная инвалидизация состояния после изменения графа;
- явная отмена без повторного применения старых effects;
- хранение состояния через `AiGameBridge`;
- русская форма длительности и тайм-аута;
- живая подсветка и панель `След ИИ`;
- отдельный `runtime:smoke`;
- отдельный Playwright-сценарий.

## Найденные и исправленные проблемы

### Строгая типизация продолжения

Production build нашёл возможное `undefined` при чтении прежнего состояния. Введено отдельное суженное `resumableState`.

### Цикл MutationObserver

Первая версия вспомогательной панели могла пересоздавать наблюдаемый DOM каждый кадр. Добавлена проверка выбранной ноды и повторное использование уже установленной панели.

### Невидимые параметры

Поля длительности сначала находились ниже скрытой технической карточки. Они перенесены прямо в основную человеко-понятную панель выбранной ноды.

### Несовпадение шага ввода и сохранения

Общий human UI сохраняет числовые значения целыми. Поля ожидания переведены на целые секунды, чтобы интерфейс не обещал дробную точность, которую затем округляет.

### Проверка реального сохранения

Playwright теперь меняет длительность с `2` на `4`, нажимает `Сохранить параметры` и проверяет значение в сохранённом графе.

## Что намеренно не сделано

- реактивные условия и автоматические abort;
- событийная шина;

- резервирование укрытий;
- сохранение execution state в сцене;
- массовый запуск для армии.
