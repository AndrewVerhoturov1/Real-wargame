## Цель

Превратить подготовленный контракт `running` в первое настоящее игровое действие, которое длится несколько тиков ИИ.

Выбрана нода:

```text
MoveToBlackboardPosition
Двигаться к позиции из памяти
```

## Принятое решение

Не двигать бойца внутри AI runtime.

Разделение:

```text
AiGraphRuntime
→ хранит цель, статус и lifecycle;

AiStatefulMoveGameBridge
→ переводит effects в MoveOrder;

SimulationTick
→ единственный изменяет координаты бойца.
```

Это не связывает core AI с PixiJS, DOM или `SimulationState`.

## Безопасность приказов

Для AI-движения введён owner token.

Главное требование:

```text
старый cancel ИИ не должен удалить новый приказ игрока
```

Добавлен отдельный bridge-smoke тест:

1. ИИ создаёт принадлежащий ему приказ;
2. игрок заменяет его своим приказом;
3. приходит очистка старого AI-token;
4. приказ игрока остаётся на месте.

## Исправления, найденные проверками

### Строгая типизация

TypeScript выявил потерю сужения типа `activeData` после проверки состояния. Исправлено явным безопасным сужением до `AiGraphMoveExecutionData` после валидатора.

### Старый editor smoke

Первое подключение использовало новое имя установщика bridge. Сохранён совместимый alias `installAiGameBridge`, чтобы не ломать существующий контракт и документацию.

### Экранная регистрация ноды

Первая версия регистрировала новую ноду только перед запуском AI Node Editor. Ревью выявило, что общий TypeScript-валидатор тогда мог считать экспортированный граф неизвестным.

Исправление:

- нода внесена в `src/core/ai/AiNodeTypes.ts`;
- экранный регистрационный модуль удалён;
- добавлен smoke-тест `validateAiGraph()` для графа движения.

### Компоновка `След ИИ`

Первый PNG показал наложение строк цели и расстояния из-за вложенных grid-блоков.

Исправление:

- каждая строка добавляется непосредственным ребёнком существующего `<dl>`;
- новый PNG открыт;
- строки читаются нормально.

## Итог

Теперь отдельный солдат может:

```text
начать движение
→ помнить одну цель
→ продолжать между тиками
→ показывать остаток пути
→ завершиться по прибытии
→ перейти к следующему шагу
→ безопасно отмениться
```

Новый приказ игрока имеет приоритет и не удаляется устаревшей очисткой ИИ.

## Ограничения

- Движение прямолинейное.
- Pathfinding пока отсутствует.
- Нет резервирования укрытий.
- Нет полной реактивной перепроверки ветки.
- Автоматически исполняется только выбранный солдат.
- Execution state не сохраняется в scene JSON.
