# Journal — Stateful AI Movement v1

Дата: 2026-07-12  
Рабочая ветка: `feature/ai-running-move-v1`  
Целевая ветка: `real-wargame-preview`  
PR: `#54 Add stateful AI movement v1`

## Цель

Превратить подготовленный контракт `running` в первое настоящее игровое действие, которое длится несколько тиков ИИ.

Выбрана нода:

```text
MoveToBlackboardPosition
Двигаться к позиции из памяти
```

## Архитектура

Боец не перемещается внутри AI runtime.

```text
AiGraphRuntime
→ хранит цель, статус и lifecycle;

AiStatefulMoveGameBridge
→ переводит runtime-effects в MoveOrder;

SimulationTick
→ единственный изменяет координаты бойца.
```

Core AI не связан с PixiJS, DOM, localStorage или `SimulationState`.

Длительные effects остаются контрактом `AiGraphRuntime`, а не мгновенного `AiGraphRunner`. Bridge принимает их только через проверяющий декодер.

## Безопасность приказов

Для AI-движения введён owner token.

Главное требование:

```text
старый cancel ИИ не должен удалить новый приказ игрока
```

Проверяются оба пользовательских формата:

- явно помеченный приказ `source: player`;
- реальный совместимый приказ правого клика без `source` и без token.

В обоих случаях устаревшая очистка старого AI-token сохраняет приказ игрока.

## Исправления по ходу проверки

### Строгая типизация состояния

TypeScript выявил потерю сужения `activeData`. После валидатора состояние явно сужается до `AiGraphMoveExecutionData`.

### Совместимость установщика bridge

Сохранён совместимый alias `installAiGameBridge`, чтобы старые smoke-проверки и интерфейсы не ломались.

### Единый каталог нод

Нода зарегистрирована в `src/core/ai/AiNodeTypes.ts`, поэтому её понимают палитра, валидатор, импорт, экспорт и runtime.

### Компоновка `След ИИ`

Первый PNG показал наложение новых строк. Строки цели, координат и остатка пути перенесены непосредственными детьми существующего `<dl>`.

### Следующий шаг последовательности

Ревью выявило, что поздняя очистка движения могла заменить подпись следующего действия на `observe`. Bridge теперь проверяет последующие effects и не скрывает, например, выполненное после прибытия действие `Лечь`.

### Настоящее перемещение

Bridge-smoke теперь не ограничивается проверкой созданного приказа: он запускает `SimulationTick` и подтверждает, что расстояние до цели действительно уменьшается.

### Реальный приказ правого клика

Добавлен тест совместимого приказа без ownership-полей. Такой приказ определяется как пользовательский и переживает очистку старого AI-token.

### Параметры новой ноды

Новая `MoveToBlackboardPosition`, добавленная из палитры, сразу сохраняет:

```text
targetKey = best_cover_position
acceptanceRadiusCells = 0.2
timeoutSeconds = 15
```

Для этого не требуется вручную открывать JSON или повторно нажимать сохранение.

## Итог

Отдельный солдат теперь может:

```text
начать движение
→ помнить одну замороженную цель
→ физически двигаться между тиками
→ показывать остаток пути
→ завершиться по прибытии
→ перейти к следующему шагу
→ безопасно отмениться
```

Новый приказ игрока имеет приоритет и не удаляется устаревшей очисткой ИИ.

## Ограничения

- Движение прямолинейное.
- Pathfinding пока отсутствует.
- Нет резервирования укрытий.
- Нет полной реактивной перепроверки ветки.
- Автоматически исполняется только выбранный солдат.
- Execution state не сохраняется в scene JSON.
