# Геометрия объектов карты и чистая проверка линии выстрела

## Состояние этапа

Работа выполняется в ветке `feature/20260722-map-object-geometry-ballistic-probe` от `real-wargame-preview` с исходным коммитом `74726d572da6560b5d2dc1e01b0a22f6b09b9599`.

Кодовая часть этапа реализована. Полная приёмка остаётся открытой до выполнения обязательных команд TypeScript, целевых smoke-проверок и production build в полноценном рабочем каталоге. GitHub Actions, Playwright, Chromium и deployment для этого этапа не запускаются.

## Каноническая семантика координат

Поля `MapObject.x` и `MapObject.y` хранят координаты якоря базовой клетки объекта. Это не верхний левый угол всего занимаемого прямоугольника и не геометрический центр многоклеточного объекта.

Геометрический центр в координатах сетки всегда равен:

```text
centerX = x + 0.5
centerY = y + 0.5
```

Размер объекта не сдвигает центр. Объект `1×1` с `x=3`, `y=4` имеет центр `(3.5, 4.5)`. Объект `4×2` с теми же координатами имеет тот же центр.

Это соглашение соответствует существующим картам, нормализации разрешения, отрисовке и положению объектов в редакторе. Миграция сохранённых карт не требуется.

## Единицы измерения

- `x`, `y`, `widthCells`, `heightCells` и границы геометрии выражаются в клетках сетки.
- Перевод в метры выполняется через `map.metersPerCell`.
- `losHeightMeters` выражается в метрах.
- Высота рельефа берётся через `sampleSmoothHeightLevel` и переводится по действующему шагу высоты `2 м`.
- Баллистические точки и расстояния выражаются в метрах.

## Канонический модуль

`src/core/map/MapObjectGeometry.ts` является общей чистой геометрией объекта карты. Он предоставляет:

- центр и обратное преобразование центра в якорь;
- размеры в клетках и метрах;
- локальные оси повёрнутого объекта;
- повёрнутые границы и ограничивающий прямоугольник;
- преобразования между локальными и мировыми координатами;
- проверку точки, окружности и прямоугольника;
- пересечение отрезка с повёрнутым объектом;
- каноническую высоту объекта.

Поворот задаётся `rotationRadians` вокруг канонического центра. Точное пересечение выполняется с ориентированным прямоугольником, а не только с его грубой осевой рамкой.

## Потребители

### Пространственный индекс

`MapObjectSpatialIndex` остаётся единственным индексом объектов карты. Он строит корзины по каноническим границам и сохраняет запросы точки, прямоугольника, окружности и отрезка. Инвалидирование по-прежнему привязано к ревизии слоя `objects`.

### Навигация

`GridNavigation` использует канонические границы, проверку точки и пересечение окружности бойца с повёрнутым объектом. Локальная проверка клетки сначала получает кандидатов из существующего индекса и не перебирает все объекты карты.

### Видимость

`VisibilityStaticGrid` растеризует объект по точному пересечению его повёрнутого прямоугольника с клеткой. Основание и верх объекта вычисляются от того же центра и сглаженного рельефа.

### Укрытия

`SmallArmsCoverEvaluation` получает кандидатов из существующего индекса и подтверждает пересечение линии угрозы с каноническим повёрнутым прямоугольником. Старая формула `x + widthCells / 2`, `y + heightCells / 2` удалена.

### Баллистика

`BallisticTrace` является чистым геометрическим ядром. Оно использует:

- сглаженный рельеф;
- существующий пространственный индекс;
- каноническое пересечение с объектами;
- `UnitHitShapes` и действующую `unit.behaviorRuntime.posture` для бойцов.

`BallisticRaycast.traceProjectile` вызывает это ядро, а затем отдельно применяет подавление и остальные боевые последствия. Геометрия реального выстрела и чистой пробы поэтому не дублируется.

## Чистая линия выстрела

`BallisticLineProbe` принимает подготовленный контекст и явные точки начала и цели. Он не читает время симуляции, не использует случайность и не изменяет состояние.

Результат различает:

- свободную линию;
- рельеф;
- объект карты;
- бойца;
- ограничение дальности.

Также возвращаются идентификатор препятствия, расстояние до первого препятствия, точка пересечения, зона попадания бойца и минимальный запас по высоте. Запас равен `0` при блокировке; для свободной линии это минимальное честно вычисленное расстояние до рельефа или верхней/нижней границы пересечённого по плану объекта. Если измеримого препятствия нет, значение может быть `null`.

## Производительность

Один баллистический запрос не сканирует все объекты карты. Худшая работа запроса состоит из:

```text
O(длина луча / 0.5 м + кандидаты MapObjectSpatialIndex + число бойцов)
```

Новых worker, глобальных кэшей, постоянной работы в каждом тике и второго индекса объектов нет. Подготовленный контекст можно переиспользовать для нескольких локальных проверок, пока карта и список бойцов соответствуют запросу.

## Проверки этапа

Добавлены:

- `npm run map-object-geometry-parity:smoke`;
- `npm run ballistic-line-probe:smoke`.

Они проверяют крупные и повёрнутые объекты, границы, соседние клетки, порядок объектов, согласие индекса, навигации, видимости, укрытий и баллистики, чистоту пробы, позу бойца, детерминизм и согласие реального выстрела с пробой.

## Ограничения

- Индекс объектов переиспользуется, но бойцы пока проверяются линейно; для малого числа локальных проб это соответствует текущей архитектуре.
- Запас линии не моделирует вероятностный разброс, проникновение и динамику снаряда: это геометрический запас прямого луча.
- Отрисовка уже использует подтверждённое соглашение `(x + 0.5, y + 0.5)` и визуально не сдвигается; игровая геометрия больше не должна выводиться из кода отрисовки.
