# Graph v2: типизированные контракты и подграфы

Дата: 2026-07-13  
Рабочая ветка: `feat/ai-graph-v2-contracts-subgraphs-2026-07-13`

## Что изменилось простыми словами

Редактор больше не считает любую связь между нодами допустимой. Каждая нода имеет единый контракт: название, русское описание, параметры, допустимое число дочерних шагов, тип исполнения и входы/выходы.

Например, позиция на карте имеет тип `position`, боец — `unitId`, событие — `event`. Редактор не разрешает передать бойца во вход позиции и объясняет причину по-русски.

## Типизированный порт

Порт — это подписанный вход или выход ноды. Его тип показывает, какое значение через него проходит:

- число;
- да/нет;
- текст;
- позиция;
- боец, объект или место;
- событие;
- план;
- маршрут.

Обычные круглые связи по-прежнему задают порядок выполнения. Подписанные порты передают данные и рисуются отдельными пунктирными линиями.

## Контракт ноды

Контракт хранится в едином реестре и используется одновременно core-runtime и редактором. В нём находятся:

- английское и русское название;
- понятное описание;
- входы и выходы;
- обязательные и необязательные параметры;
- допустимые диапазоны и варианты;
- политика дочерних нод;
- тип жизненного цикла.

Значения по умолчанию больше не должны независимо копироваться в разных UI-файлах.

## Graph v2 и старые графы

Graph v1 продолжает загружаться. Кнопка **«Проверить и обновить формат графа»** выполняет миграцию в памяти, затем строгую проверку и только после успеха сохраняет Graph v2.

Миграция:

- сохраняет порядок детей;
- переводит параметры в типизированный формат;
- добавляет схему Blackboard и ссылки на подграфы;
- переносит незнакомые старые поля в `legacyMetadata`;
- не перезаписывает старый файл при ошибке;
- повторно даёт тот же результат.

Для автоматизации существует `npm run ai-graph:migrate -- input.json output.json`, но обычному пользователю терминал не нужен.

## Области памяти

Память разделена на пять уровней:

1. **Постоянная память бойца** — знания, которые должны пережить смену текущего действия.
2. **Память runtime session** — рабочие данные текущего сеанса ИИ.
3. **Память активного состояния** — очищается при выходе из состояния.
4. **Локальная память подграфа** — видна только конкретному вызову подграфа.
5. **Локальное состояние ноды** — прогресс конкретной длительной ноды.

Подграф не может изменить память родителя без явного выходного binding. В панели «След ИИ» можно раскрыть области памяти и увидеть, к какой области относится каждый ключ.

## Подграфы

Подграф — переиспользуемый небольшой граф поведения с явными входами, выходами и изолированной памятью.

Добавлены четыре шаблона:

- `Занять укрытие`;
- `Перезарядить оружие`;
- `Реагировать на обстрел`;
- `Двигаться и наблюдать`.

Подграф выбирается в инспекторе. Двойной клик открывает его содержимое. Сверху показывается цепочка навигации и кнопка возврата к родительскому графу.

Активный подграф сохраняется вместе со сценой. После загрузки длительное действие продолжается через `update`, а не запускается второй раз через `start`. Отмена родителя отменяет вложенное действие и выполняет cleanup один раз. Прямая и косвенная рекурсия запрещена.

## Управляющие ноды этапа 2

Перед Graph v2 завершён минимально необходимый набор:

- `WaitForEvent` — ждёт выбранное событие и при необходимости поглощает его;
- `Timeout` — ограничивает время дочерней ветви и безопасно её отменяет;
- `Retry` — повторяет провалившуюся ветвь не больше указанного числа попыток.

## Проверка графа

Панель ошибок различает `error`, `warning` и `info`. Сообщение содержит технический код, ноду, параметр или порт, русское объяснение и способ исправления. Клик по сообщению выделяет нужную ноду.

Проверяются параметры, диапазоны, enum, порты, число детей, обязательные входы, недостижимые ноды, неиспользуемые выходы, циклы, неизвестные типы и рекурсивные подграфы.

## Контрольный сценарий

Автоматическая проверка воспроизводит цепочку:

```text
приказ движения
→ MoveTo Running
→ shot_nearby
→ отмена старого движения
→ подграф «Занять укрытие»
→ сохранение во время движения
→ восстановление без повторного start
→ достижение укрытия
→ подграф «Двигаться и наблюдать»
```

Она также проверяет `ownerToken`, отсутствие зависшего execution state, изоляцию локальной памяти и полный trace-путь.

## Что проверить вручную

1. Открыть старый Graph v1 и увидеть предупреждение.
2. Нажать «Проверить и обновить формат графа» и убедиться, что появляется Graph v2.
3. Создать несовместимую связь и увидеть русское объяснение.
4. Проверить подписи типов портов и диапазоны параметров в инспекторе.
5. Добавить `Подграф поведения`, выбрать шаблон и открыть его двойным кликом.
6. Проверить цепочку «Главный граф → подграф» и возврат назад.
7. Запустить проверку графа, кликнуть по ошибке и убедиться, что выделена нужная нода.
8. В живом runtime проверить активную ноду, активный подграф, путь и области памяти.
9. Сохранить сцену во время движения внутри подграфа и убедиться, что после загрузки движение продолжается без нового приказа.
