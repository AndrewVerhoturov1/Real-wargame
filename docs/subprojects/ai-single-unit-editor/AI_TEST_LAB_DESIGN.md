# Полигон ИИ — проект игровой испытательной сцены

Дата: 2026-07-10  
Ветка: `real-wargame-preview`

## Цель

Превратить текущий редактор карты в управляемую испытательную сцену, где можно вручную задать бойца, источник опасности, свойства укрытий и пошагово проверить решение `GraphRunner`.

## Ограничения

- `main` не изменять.
- Сохранить старые круглые и прямоугольные зоны опасности.
- Не добавлять настоящую баллистику, пули, ранения и вражеский боевой ИИ.
- Не возвращать старые одноразовые AI-ноды.
- Все значения, которые видит AI-граф, должны происходить из состояния сцены, а не из постоянных заглушек.

## 1. Боец

У выбранного бойца редактируются:

- имя и профиль;
- стойкость, осторожность, решительность, дисциплина, инициатива, тактика, владение оружием;
- усталость, мораль, замешательство, здоровье, внимание, зрение, интуиция, скорость и скрытность;
- стресс, подавление, патроны и готовность оружия;
- дальность и угол обзора;
- обычная скорость движения.

Базовая скорость пехотинца меняется с `2.2 клетки/с` на `0.5 клетки/с`. При масштабе 10 м на клетку это 5 м/с — быстрый боевой бег, а не прежние 22–23 м/с.

## 2. Направленная угроза

Старые зоны остаются. Дополнительно зона может работать как направленный источник огня со свойствами:

- направление;
- угол сектора;
- дальность;
- минимальная дальность;
- сила опасности;
- подавление и стресс в секунду;
- падение силы с расстоянием;
- включена/выключена;
- источник виден и/или известен бойцу.

Расчёт учитывает положение бойца внутри сектора, расстояние, позу и защиту укрытия между угрозой и бойцом. Несколько угроз складываются с ограничением 100.

## 3. Укрытия

Каждый предмет карты получает необязательные свойства:

- физическая защита 0–100;
- маскировка 0–100;
- простреливаемость;
- минимальная поза, которую предмет способен закрыть.

Значения по умолчанию зависят от типа предмета, но могут быть изменены в редакторе. Защита применяется только когда предмет расположен между источником огня и бойцом.

## 4. Испытание

В игре добавляется панель:

- пауза;
- один расчёт AI;
- расчёт и применение решения;
- один шаг симуляции;
- сброс выбранного бойца;
- сброс всей сцены;
- множители времени `×0.25`, `×0.5`, `×1`, `×2`, `×4`, `×10`.

Один расчёт без применения публикует trace, но не двигает бойца. Это позволяет изучить выбор `UtilitySelector` на паузе.

## 5. Диагностика

Для выбранного бойца показываются:

- суммарная опасность, подавление и прирост стресса;
- главный источник угрозы, расстояние и направление;
- найденное укрытие и защита от текущей угрозы;
- патроны и готовность оружия;
- последняя выбранная ветка и объяснение.

На карте направленная угроза рисуется сектором со стрелкой.

## 6. Сохранение

Экспорт сцены обновляется до следующей версии и сохраняет:

- индивидуальные характеристики бойцов;
- текущее испытательное состояние бойцов;
- свойства направленных угроз;
- свойства укрытий.

Старые JSON без новых полей продолжают загружаться с безопасными значениями по умолчанию.

## 7. Проверка

Обязательные проверки:

- расчёт сектора и падения силы;
- сумма нескольких угроз;
- защита только с правильной стороны;
- реальные значения `ammo`, `weaponReady`, `enemyVisible`, `enemyKnown`, `current_target` в blackboard;
- скорость по умолчанию 0.5 клетки/с;
- наличие режима ×10;
- экспорт и обратная загрузка новых полей;
- `npm run build`, `npm run editor:smoke`, `npm run engine:smoke`, `npm run validate:ai-graph`, новый smoke полигона.
