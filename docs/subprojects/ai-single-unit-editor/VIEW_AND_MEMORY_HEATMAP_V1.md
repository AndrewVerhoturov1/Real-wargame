# Обзор и память — поклеточная карта видимости v1

## Назначение

Слой **«Обзор и память»** показывает субъективные возможности выбранного бойца:

- поклеточная тепловая карта отображает качество текущего наблюдения местности;
- текущие и старые контакты показываются отдельными метками;
- старый обзор местности не сохраняется как призрачная тепловая карта;
- слой не раскрывает скрытые объекты и не заменяет фактическое обнаружение цели.

## Главное изменение относительно Perception and Attention v1

Физическое движение узкого сектора внимания удалено.

- На марше боец смотрит в направлении движения или корпуса и не сканирует пространство вращением.
- В режиме наблюдения используется стабильное широкое распределение внимания.
- В режиме поиска назначается неподвижный сектор повышенного внимания.
- При стрельбе внимание удерживается около цели.
- Боковой и задний контроль моделируется весами и интервалами проверки, а не анимацией головы.

Поля старых сцен `scanDirection`, `scanProgress01` и `scanSpeedDegreesPerSecond` принимаются для совместимости, но больше не создают физическое сканирование.

## Что означает цвет клетки

Каждая клетка получает качество наблюдения от `0` до `255` в компактном `Uint8Array`.

На качество влияют:

1. прямая геометрическая видимость;
2. складки рельефа и высота глаз бойца;
3. здания, камни, укрытия и другие предметы с высотой;
4. накопленное ослабление обзора лесом;
5. физическая дистанция в метрах;
6. направление и режим внимания;
7. усталость, здоровье, замешательство и подавление наблюдателя.

Качество клетки не учитывает скрытые свойства конкретной цели. Поза, движение, стрельба, размер и маскировка цели применяются отдельно только при обработке реального стимула.

## Дальность зрения

Профиль бойца хранит:

- `maximumVisualRangeMeters` — практическую максимальную дальность;
- `distanceFalloffStartMeters` — дистанцию начала ухудшения;
- `distanceFalloffExponent` — крутизну падения качества;
- `detectionVariancePercent` — диапазон небольшой стабильной случайности обнаружения.

До начала падения дистанционный коэффициент равен `1`. Затем качество плавно уменьшается и достигает нуля на максимальной дальности. Резкой границы идеальной видимости больше нет.

## Обнаружение цели во времени

Фактический контакт по-прежнему строится только для существующих стимулов. Для каждой цели используется точный LOS, а не перебор всех клеток тепловой карты.

Скорость накопления признаков зависит от:

- качества наблюдения направления и дистанции;
- проходимости линии обзора;
- позы цели;
- движения цели;
- действия цели, включая выстрел;
- размера и маскировки;
- состояния и способностей наблюдателя;
- небольшого стабильного множителя обнаружения.

Случайный множитель:

- создаётся детерминированно из `observerId`, `stimulusId` и номера эпизода;
- по умолчанию ограничен диапазоном `0,90…1,10`;
- не меняется каждый кадр;
- не зависит от FPS;
- сохраняется в контакте и сцене;
- для старой сцены без этого поля получает нейтральное значение `1`.

Поэтому заметная цель обычно обнаруживается быстрее, слабозаметная — медленнее, но одинаковые условия не обязаны всегда давать одну и ту же секунду обнаружения.

## Память

Исторические сведения остаются существующими метками контактов:

- признак;
- подозрение;
- звуковой контакт;
- последнее известное положение;
- опознанная или подтверждённая цель;
- область неопределённости.

После потери наблюдения уверенность снижается, а область неопределённости растёт. Тепловая карта показывает только текущий обзор и не хранит старое состояние местности.

## Оптимизация

### Нулевые расходы скрытого слоя

Поле не строится, когда:

- слой выключен;
- отключён переключатель текущего обзора;
- боец не выбран;
- открыт режим редактирования карты.

### Статический кеш карты

Рельеф, высоты предметов, лес и блокирующие признаки переводятся в компактные типизированные массивы. Кеш пересобирается только при изменении визуальной ревизии карты.

### Кеш поля бойца

Ключ расчёта содержит:

- ID бойца;
- квантованную позицию;
- позу;
- режим и направление внимания;
- параметры профиля;
- метрические настройки зрения;
- визуальную ревизию карты;
- разрешение сетки.

Движение курсора и камеры не входит в ключ и не вызывает перестроение.

Незначительное перемещение меньше четверти клетки использует прежнее поле. Во время движения новое поле строится не чаще одного раза примерно в `0,2` секунды.

### Радиальный расчёт

Карта строится лучами к периметру ограниченной области, а не отдельным полным LOS для каждой клетки. На луче накапливаются горизонт рельефа, высота предметов и проходимость леса.

### Один растровый спрайт

PixiJS не создаёт объект на каждую клетку. `Uint8Array` переводится в один Canvas/Texture и выводится одним `Sprite` с nearest-neighbor масштабированием. Текстура загружается только при изменении ревизии поля. Метки памяти рисуются отдельным `Graphics` поверх неё.

## Диагностика

В панели доступны:

- число перестроений;
- число попаданий в кеш;
- причина последнего обновления;
- количество обработанных клеточных шагов.

Для автоматической проверки публикуется `window.__realWargameViewMemoryDebug` с представлением рендера, количеством загрузок текстуры, меток и перестроений поля.

## Автоматические проверки

```text
npm run view-memory-heatmap:smoke
npm run view-memory-heatmap-performance:smoke
npm run perception:smoke
npm run perception-variance:smoke
npm run perception-performance:smoke
npm run attention-ai-nodes:smoke
npm run workspace:smoke
npm run game-editor:smoke
npm run build
npm run docs:check
```

Проверяются:

- отсутствие физического сканирования;
- тень за зданием;
- ослабление лесом;
- падение качества с дистанцией;
- инвалидирование по ревизии карты;
- нулевой расчёт скрытого слоя;
- компактное хранение одним байтом на клетку;
- кеш неподвижного бойца;
- ограничение частоты перестроений при движении;
- отсутствие точных LOS из расчёта тепловой карты;
- стабильность случайности и независимость от FPS;
- обратная совместимость памяти;
- один растровый спрайт вместо клеточных объектов.

## Честные ограничения v1

- Поле строится только для выбранного бойца.
- Расчёт выполняется синхронно, но строго кешируется и ограничивается по частоте; отдельная фоновая задача может понадобиться для карт значительно крупнее текущих.
- Полноценная сторона противника с большим количеством боевых юнитов ещё не завершена.
- Командный обмен контактами, оптика, ночь и подробная погода не входят в эту версию.
- Тепловая карта является диагностическим пользовательским слоем и не должна использоваться ИИ как источник объективного всезнания.
