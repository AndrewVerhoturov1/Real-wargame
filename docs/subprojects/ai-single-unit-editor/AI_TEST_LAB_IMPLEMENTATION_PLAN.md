# AI Test Lab Implementation Plan

> **Для исполнителя:** внедрять по задачам, после каждой задачи запускать подходящий smoke/build и делать отдельный коммит.

**Цель:** сделать игровую сцену полноценным полигоном для ручной настройки бойца, угроз и укрытий с пошаговой проверкой GraphRunner.

**Архитектура:** математика угроз и укрытий живёт в отдельных core-модулях без DOM/Pixi. `SimulationState` хранит только данные и команды изменения. Интерфейс редактора вызывает команды состояния, рендерер только отображает сектор и диагностику, `AiGameBridge` строит blackboard из общих расчётов.

**Стек:** TypeScript, Vite, PixiJS 7, Node.js smoke-скрипты.

## Общие ограничения

- Работать только в `real-wargame-preview`.
- Не менять `main`.
- Старые JSON и обычные зоны должны продолжить работать.
- Скорость бойца по умолчанию: `0.5` клетки/с.
- Множители времени: `0.25`, `0.5`, `1`, `2`, `4`, `10`.
- Не делать визуальную проверку без настоящего запуска браузера.
