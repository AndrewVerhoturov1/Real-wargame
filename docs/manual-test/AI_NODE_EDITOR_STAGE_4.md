# Manual test — AI Node Editor Stage 4 Authoring

Дата: 2026-07-09  
Ветка: `real-wargame-preview`  
Назначение: проверить, что редактор нод уже не только видимый, но умеет менять рабочий граф и удобно работать на canvas.

## Запуск

Запустить двойным кликом:

```text
Run-AI-Node-Editor.bat
```

Откроется:

```text
http://127.0.0.1:5173/ai-node-editor.html
```

## Быстрая проверка

1. Нажать сверху:

```text
Auto 4–5
```

Ожидание:

```text
Point 4 OK / Пункт 4 OK
Point 5 OK / Пункт 5 OK
```

2. Если автопроверка прошла, перейти к authoring-проверке ниже.

## Authoring-проверка

### 1. Добавить ноду

Открыть левую панель `+ Add node`. В палитре нажать любую ноду, например:

```text
Danger Above Threshold
```

Ожидание:

```text
новая нода появилась примерно в центре текущего вида;
она выбрана;
справа открыт её инспектор.
```

### 2. Перетащить ноду

Зажать новую ноду мышью, перетащить и отпустить.

Ожидание:

```text
нода переместилась;
линии графа обновились;
позиция запомнилась в браузере.
```

### 3. Передвигать canvas

Зажать пустое место поля редактора и потянуть.

Ожидание:

```text
весь canvas двигается;
ноды и линии двигаются вместе.
```

### 4. Масштабировать canvas

Покрутить колесо мыши над полем редактора или нажать:

```text
+
-
100%
Fit
```

Ожидание:

```text
ноды масштабируются вместе с линиями;
Fit показывает граф удобнее;
100% возвращает обычный масштаб.
```

### 5. Связать ноды протягиванием

У каждой ноды справа есть маленькая жёлтая точка-порт.

Действие:

```text
зажать правую точку одной ноды;
потянуть линию к другой ноде;
отпустить на другой ноде.
```

Ожидание:

```text
появилась связь parent → child;
линия осталась на графе;
в инспекторе parent появилась новая child-связь.
```

Fallback-способ через список `Link selected → child` можно оставить, но главный способ — протягивание из точки.

### 6. Контекстное меню

Правый клик по ноде.

Ожидание:

```text
открывается меню;
есть Select, Add child, Duplicate, Set as link source, Link source → this, Center view, Unlink all children, Delete.
```

### 7. Изменить параметры

В инспекторе изменить, например:

```json
{
  "threshold": 45
}
```

Нажать:

```text
Save node
```

Ожидание:

```text
сообщение Saved node;
Graph JSON preview снизу показывает новое значение.
```

### 8. Проверить изменённый граф через local engine

Нажать:

```text
Validate
```

Ожидание:

```text
validation.valid = true
```

Если граф сломан, ожидаемая нормальная ошибка — русско/английское сообщение с nodeId.

### 9. Экспортировать JSON

Нажать:

```text
Export
```

Ожидание:

```text
браузер скачал soldier_default_survival_graph.json или soldier_custom_graph.json;
в файле есть английский base и русский overlay: displayName + displayNameRu.
```

### 10. Импортировать JSON обратно

Нажать:

```text
Import
```

Выбрать экспортированный файл.

Ожидание:

```text
граф загрузился обратно;
ноды отображаются;
Validate всё ещё проходит.
```

### 11. Сбросить к базовому графу

Нажать:

```text
Reset
```

Ожидание:

```text
граф вернулся к bundled Soldier Survival Graph.
```

## Что этот этап ещё НЕ делает

```text
не подключает граф к живому SimulationTick;
не двигает настоящего солдата на карте;
не сохраняет JSON прямо в repo-файлы;
не делает WebSocket;
не делает worker_threads.
```

## Что прислать в чат при ошибке

1. Текст из окна `Run-AI-Node-Editor.bat`.
2. Что написано внизу в `Validation / Engine result`.
3. Что именно делал: add / drag / pan / zoom / port-link / context-menu / save / export / import.
4. Если ошибка после `Validate`, прислать JSON ошибки из нижнего окна.
