# Ctrl Cell Inspector Design

## Goal

При удержании `Ctrl` над картой показывать чёткую HTML-плашку с главной информацией о клетке. Содержимое зависит от активного слоя правого инспектора и всегда использует субъективные данные выбранного бойца.

## Scope

Поддерживаемые режимы:

- `info` — поверхность, растительность, высота и проходимость;
- `danger` — опасность, подавление, защита, уверенность и главная понятная причина;
- `positions` — найденная тактическая позиция в клетке и её ключевые показатели;
- `stealth` — маскировка, заметность, безопасность и отличие маскировки от пулевой защиты;
- `memory` — качество текущего обзора либо находящийся в клетке контакт из памяти;
- `routeCost` — стоимость клетки и три крупнейших вклада в неё. До переноса слоя в правый инспектор этот режим доступен, когда слой стоимости включён при открытой вкладке «Инфо».

## Interaction

- Плашка появляется только пока нажат `Ctrl` и указатель находится над холстом карты.
- Плашка размещается рядом с указателем и ограничивается границами окна.
- Отпускание `Ctrl`, уход указателя с карты, потеря фокуса окна или переход в редактор скрывают плашку.
- Пока `Ctrl` удерживается, содержимое обновляется при переходе в другую клетку и редким лёгким опросом готовых снимков, чтобы отобразить результат фонового расчёта без движения мыши.
- Плашка не перехватывает ввод мыши.

## Data ownership

UI не строит поля и не запускает расчёты:

- опасность и скрытность читаются из готового `PreparedAwarenessWorldSnapshot` сервиса тактических позиций;
- обзор читается только из уже закэшированного поля видимости;
- стоимость маршрута читается только из уже готового результата worker;
- позиции читаются из опубликованного списка кандидатов;
- физические свойства читаются напрямую из клетки карты и активного профиля материалов.

Если готового снимка нет, плашка сообщает «данные подготавливаются» и не запускает расчёт сама.

## Presentation

Плашка содержит:

1. короткий заголовок слоя и оценку;
2. словесный уровень;
3. одну-две главные причины;
4. не более четырёх ключевых показателей;
5. при необходимости примечание об ограниченной уверенности или отсутствии известных угроз.

Маскировка и защита от пуль показываются раздельно. Неизвестные бойцу угрозы и контакты не раскрываются.

## Performance review

- hot path: движение указателя при удержании `Ctrl` и опрос раз в 250 мс;
- worst-case complexity: `O(1)` для клеточных массивов, `O(C)` для малого списка контактов или кандидатов, где `C` уже ограничен системой;
- main-thread work: чтение нескольких значений и обновление одной DOM-плашки только при изменении ключа содержимого;
- full-map work: отсутствует;
- shared prepared result: awareness worker field, cached visibility field, ready route-cost field, published tactical candidates;
- invalidation identity: ключ содержит слой, клетку, выбранного бойца и идентичности готовых снимков;
- worker/queue budget: новые worker и очереди не создаются;
- cache memory bound: новый кэш полей не создаётся;
- teardown: снимаются все обработчики, таймер останавливается, DOM-узел удаляется.

## Verification

- статический smoke-контракт проверяет `Ctrl`, симметричный teardown и запрет вызовов, запускающих тяжёлые расчёты;
- TypeScript;
- `workspace:smoke`;
- `navigation-overlay:smoke`;
- новый `cell-inspector:smoke`;
- production build;
- визуальная проверка готовится отдельно и запускается только после явного разрешения пользователя.
