# Сосредоточенное внимание к подозрительному контакту

## Цель

Когда у бойца есть субъективная предполагаемая позиция противника, но сам противник сейчас не виден, Graph v2 должен уметь направить существующий режим поиска на эту позицию. Сосредоточенный поиск использует обычный настраиваемый профиль внимания, поэтому прямой, периферийный и тыловой обзор продолжают работать по настройкам профиля.

## Границы

- Не создаётся новая система восприятия, памяти или поведения.
- Не изменяются правила обнаружения контактов.
- Не добавляется новая нода: расширяется существующая `SetSearchSector`.
- Не затрагиваются стрельба, подавление, движение по маршруту и физические действия.
- Встроенный чистый Graph v2 остаётся чистым рабочим полотном.
- `real-wargame-preview` и `main` не изменяются.

## Решение

`SetSearchSector` получает необязательный источник направления:

- `fixed` — прежний фиксированный угол `centerDegrees`;
- `blackboard_position` — направление от позиции из `originPositionKey` к позиции из `targetPositionKey`.

Значения по умолчанию для динамического источника:

- `originPositionKey = self_position`;
- `targetPositionKey = suspected_enemy_position`.

Перед выполнением старого вычислителя `AiGraphRunner` в постоянное время вычисляет угол через `atan2`. Если одна из позиций отсутствует или совпадает с другой, нода корректно проваливается, позволяя `Selector` перейти к ветви автоматического внимания.

При применении сектора:

- направление внимания всегда переводится на рассчитанный сектор;
- неподвижный боец, получивший сектор от AI, разворачивает корпус к центру сектора;
- движущийся боец сохраняет направление корпуса по маршруту, но смотрит в сторону подозрения;
- сектор, заданный игроком, не разворачивает корпус автоматически.

## Импортируемый пример временной ветки

Встроенный граф остаётся чистым. Для проверки добавлен отдельный файл:

`public/ai-examples/suspected-contact-attention.json`

Его можно импортировать в редактор. Пример выполняет:

1. уверенность лучшего контакта выше 15;
2. контакт сейчас не виден;
3. установить сектор поиска по `suspected_enemy_position`;
4. иначе вернуть автоматическое внимание.

Параметры качества, ширины и частоты проверок остаются в существующем редакторе профилей внимания.

## Совместимость

Сохранённые ноды без `centerSource` продолжают использовать `centerDegrees`. Новые параметры являются необязательными и сохраняются Graph v2. Единственное осмысленное уточнение существующей семантики: неподвижный боец разворачивается к сектору, который AI явно приказал осмотреть; движение и игрок сохраняют владение направлением корпуса в своих случаях.

## Производительность

- горячий путь: один запуск ноды Graph v2;
- сложность: `O(1)` на решение бойца;
- работа по всей карте: отсутствует;
- новые очереди, таймеры, рабочие потоки и кэши: отсутствуют;
- используются уже опубликованные субъективные значения Blackboard;
- интерфейс не вычисляет игровые факты.

## Проверка

- фиксированный сектор продолжает выдавать прежний эффект;
- сектор по позиции вычисляет правильный нормализованный угол;
- отсутствующая позиция переводит выбор на запасную ветвь;
- неподвижный AI-боец разворачивается, движущийся — нет;
- интерфейс показывает источник направления и ключи памяти;
- импортируемый Graph v2 проходит проверку и выбирает правильное направление;
- чистое стартовое полотно сохраняется;
- TypeScript, профильные проверки и производственная сборка проходят на точном HEAD.
