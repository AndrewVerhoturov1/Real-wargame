# Слой стоимости маршрута в правом инспекторе

## Цель

Перенести управление слоем стоимости маршрута из нижней панели выбранного бойца и выпадающего меню «Вид» в правый инспектор тактической карты. Новый пункт должен находиться рядом с вкладками «Опасность» и «Скрытность» и использовать существующее состояние и рендерер слоя без второй независимой реализации.

## Выбранный вариант

Добавить отдельную вкладку правого инспектора «Стоимость маршрута». Внутри неё разместить:

- переключатель отображения слоя;
- выбор вида стоимости: «Базовая местность» или «Итоговая стоимость»;
- краткое пояснение, что слой диагностический и зависит от выбранного бойца для итоговой стоимости.

Из нижней панели удалить кнопку «Карта стоимости». Из меню «Вид» удалить отдельный блок управления стоимостью, чтобы у слоя осталась одна точка управления.

Сведения о текущем маршруте, профиле, цене, длине и причине в раскрывающемся блоке нижней панели остаются: это диагностика выбранного маршрута, а не управление картографическим слоем.

## Архитектура

- `RouteCostOverlayState` остаётся единственным владельцем состояния слоя.
- `PixiRouteCostOverlayRenderer` остаётся единственным рендерером и не меняет расчёты, кеши или порядок слоёв.
- `TacticalWorkspaceBase` получает новую вкладку и контейнер для её содержимого.
- `RouteCostOverlayUi` больше не создаёт собственный блок в меню «Вид» и не подписывается на нижнюю кнопку. Он подключается к контейнеру новой вкладки правого инспектора и синхронизирует элементы с существующим состоянием.
- При переходе на вкладку слой не включается автоматически. Пользователь явно включает его переключателем.
- Переключение других вкладок не выключает слой автоматически: вкладка управляет слоем, но не становится его владельцем.

## Поведение

1. Пользователь открывает вкладку «Стоимость маршрута».
2. Видит текущее состояние слоя и выбранный режим.
3. Нажатие переключателя вызывает существующий `toggleRouteCostOverlay`.
4. Изменение режима вызывает существующий `setRouteCostOverlayMode`.
5. Нижняя панель больше не содержит кнопку карты стоимости.
6. В меню «Вид» больше нет второго набора этих же элементов.

## Производительность

- Горячий путь не меняется.
- Сложность расчёта поля и число полных проходов по карте не меняются.
- Новый код только читает готовое состояние и обновляет два элемента DOM с существующим интервалом.
- Канонический подготовленный результат, ключи сброса кеша, очереди, память и освобождение Pixi-ресурсов не меняются.
- UI не запускает новый расчёт и не становится владельцем игровых данных.

## Проверка

Обновить или добавить узкую проверку интерфейсного договора, которая подтверждает:

- в нижней панели нет `route-cost-quick-toggle`;
- в меню «Вид» не создаётся блок стоимости;
- в правом инспекторе есть вкладка и контейнер слоя стоимости;
- `RouteCostOverlayUi` использует существующие функции состояния;
- TypeScript проходит;
- `ui-compact-route-controls:smoke`, `navigation-overlay:smoke`, `workspace:smoke` и производственная сборка проходят.

Визуальная проверка браузером не входит в текущую задачу и не запускается без отдельного разрешения.
