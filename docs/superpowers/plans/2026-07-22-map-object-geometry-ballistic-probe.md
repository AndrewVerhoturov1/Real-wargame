# Каноническая геометрия объектов и чистая линия выстрела

## Подтверждённое соглашение

`MapObject.x` и `MapObject.y` — координаты якоря базовой клетки. Геометрический центр объекта всегда равен `(x + 0.5, y + 0.5)` и не зависит от `widthCells`/`heightCells`. Это соглашение подтверждено нормализацией разрешения, отрисовкой, пространственным индексом, навигацией и существующей smoke-проверкой индекса.

Обнаружены два отклонения:

1. старая баллистика вычисляет центр как `x + widthCells / 2`, `y + heightCells / 2`;
2. редактор при новом размещении вычитает половину размеров из точки указателя, хотя сохранение и отрисовка ожидают якорь базовой клетки.

Миграция данных не требуется: существующие карты читаются и рисуются по единому соглашению. Исправление редактора применяется только к новым объектам.

## План

1. Добавить чистый `MapObjectGeometry` с центром, размерами, осями, повёрнутыми границами, пересечениями точки/окружности/прямоугольника/отрезка.
2. Перевести существующий `MapObjectSpatialIndex`, навигационную растеризацию и статическую сетку видимости на эти функции.
3. Исправить размещение новых объектов редактором без изменения сохранённых карт.
4. Выделить чистый `BallisticTrace`, который использует сглаженный рельеф, каноническую геометрию и существующий индекс объектов.
5. Оставить в `BallisticRaycast` только применение боевых побочных эффектов после чистого результата.
6. Добавить `BallisticLineProbe` с явными точками начала/цели, отдельными причинами блокировки и честным запасом по высоте.
7. Добавить целевые smoke-проверки геометрического равенства, отсутствия побочных эффектов, детерминизма и согласия с реальным выстрелом.
8. Обновить архитектурную документацию и команды package.json.

## Влияние на производительность

- горячий путь: локальная проверка одного баллистического луча;
- худшая сложность: `O(длина луча / 0.5 м + кандидаты пространственного индекса + число бойцов)`;
- полного обхода объектов карты на луч нет;
- индекс и его кэш остаются существующими, ключ инвалидирования — ревизия слоя `objects`;
- новых worker, очередей, глобальных кэшей и постоянной работы в тике нет;
- полная растеризация навигации/видимости остаётся подготовительной работой при изменении соответствующей ревизии карты;
- интерфейс и отрисовка не становятся владельцами игровой геометрии.

## Проверка

Целевые команды: TypeScript, существующие smoke-проверки индекса/видимости/боя, новые проверки геометрии и пробы, production build. GitHub Actions, Playwright, Chromium и deployment не запускаются.
