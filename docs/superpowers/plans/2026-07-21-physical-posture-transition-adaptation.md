# Physical Posture Transition Adaptation Plan

**Goal:** перенести замысел сериализуемой длительной смены позы из старой ветки в актуальную архитектуру тактических позиций, движения и боя без слияния устаревших файлов.

## Архитектурные решения

- Единственная действующая поза остаётся `unit.behaviorRuntime.posture`.
- Активное физическое действие хранится прямо в `behaviorRuntime`, имеет версию, владельца и идентификатор владения и входит в экспорт сцены.
- Составные переходы меняют каноническую позу только при достижении стадии `crouched`.
- `SimulationTickLegacy` после планировщика ИИ продвигает смену позы по точному интервалу шага, затем передаёт бою и движению только неиспользованный остаток времени.
- Приказы движения, тактические позиции, ручные команды и существующий эффект Graph v2 используют одну команду смены позы; новых узлов Graph v2 не добавляется.
- Оружие проверяет конфликт при запросе огня/перезарядки, а команда позы проверяет обратный конфликт.

## Шаги

1. Перенести и расширить проверки смены позы, маршрута, владения, боя, тактического прибытия, сохранения и прямых записей позы.
2. Добавить типы, нормализацию, сериализацию, длительности, стадии и часы физического действия.
3. Добавить действие в `BehaviorModel`, загрузку/сброс в `UnitModel`, экспорт в `SceneExport`.
4. Подключить бюджет времени к `SimulationTick` и `SimulationTickLegacy`; убрать мгновенную смену позы из `MovementRuntime`.
5. Перевести `RoutedMoveOrders`, тактические приказы/занятие, ручные элементы управления и существующий эффект Graph v2 на общую команду.
6. Добавить двусторонние запреты боя и безопасную отмену при потере боеспособности/замене приказа.
7. Выполнить указанные проверки, production build, проверить diff, создать один осмысленный коммит и отправить только целевую ветку с `[skip ci]`.

## Производительность

- Горячий путь: одна O(1) проверка и продвижение действия на бойца за шаг симуляции.
- Сложность: O(число бойцов), без обходов карты и без новых выделений больших массивов.
- Рабочие потоки, очереди и кэши не добавляются.
- Проверка: типы, профильные дымовые проверки и production build.
