# Чеклист проверки Pull Request

Этот чеклист помогает Codex проверять PR перед тем, как человек решит принимать его или нет.

## Базовая проверка

- [ ] PR решает одну понятную задачу.
- [ ] PR создан из отдельной ветки.
- [ ] PR не меняет `main` напрямую.
- [ ] Есть описание изменений.
- [ ] Понятно, какие файлы изменены.
- [ ] Есть инструкция, как проверить результат.
- [ ] Указано, какие проверки запускались.
- [ ] Указано, что не проверялось.

## Проверка безопасности

- [ ] Нет секретов, токенов, паролей, ключей.
- [ ] Нет `.env` или приватных данных.
- [ ] Нет случайного удаления важных файлов.
- [ ] Нет больших изменений "заодно".
- [ ] Нет переписывания архитектуры без разрешения.
- [ ] Нет auto-merge или попытки обойти решение человека.

## Проверка scope

- [ ] Изменены только разрешённые файлы.
- [ ] Нет лишних форматирований всего репозитория.
- [ ] Нет непрошеных новых инструментов или workflow.
- [ ] Нет скрытых побочных изменений.

## Если PR пришёл из Q-режима

- [ ] Внешний исполнитель указал repository.
- [ ] Внешний исполнитель указал branch.
- [ ] Внешний исполнитель указал PR number/link.
- [ ] Внешний исполнитель перечислил changed files.
- [ ] Внешний исполнитель указал checks run.
- [ ] Внешний исполнитель указал not checked.
- [ ] Внешний исполнитель указал risks.
- [ ] Внешний исполнитель указал human verification steps.
- [ ] Если PR не создан, указана причина `PR not created: ...`.

## Если PR пришёл из X-режима

- [ ] Проверен authoritative truth order: diff/commits первичны, отчёт исполнителя вторичен (см. docs/ai/ZWORKER_MODES.md).

## Проверка отчёта PR

- [ ] Автор написал, что сделал.
- [ ] Автор написал, как проверить.
- [ ] Автор честно указал, что не проверялось.
- [ ] Автор указал риски.
- [ ] Понятно, нужна ли локальная проверка человеком.

## Вывод для человека

Codex должен дать рекомендацию одним из вариантов:

```text
Можно принимать.
Нужно доработать.
Лучше отклонить.
Нужно разделить PR.
Нужна локальная проверка перед решением.
```

## Шаблон короткого вывода

```text
PR #N: [краткое название]

Что изменилось:
- ...

Риски:
- ...

Проверки:
- ...

Что человеку посмотреть:
- ...

Рекомендация:
- принять / доработать / отклонить / проверить локально
```
