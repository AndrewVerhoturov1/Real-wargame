# Workflow OpenCode как рук Codex

OpenCode используется как технический исполнитель для ограниченных задач.

## Главная роль

OpenCode — не самостоятельный архитектор и не главный принимающий решения. Он помогает быстро выполнить понятную техническую операцию.

Формула:

```text
Codex думает, контролирует и объясняет.
OpenCode выполняет ограниченные технические поручения.
```

## Хорошие задачи для OpenCode

- Найти файл или строку.
- Проверить, где используется имя.
- Сделать простую замену.
- Запустить команду.
- Собрать ошибки в отчёт.
- Проверить список изменённых файлов.
- Обновить небольшой документ.
- Сделать маленький патч по точной инструкции.

## Плохие задачи для OpenCode

- Выбрать архитектуру.
- Переписать большой модуль.
- Самостоятельно решить, что правильно для проекта.
- Мержить PR.
- Менять `main`.
- Делать “всё сразу”.

## Как формулировать поручение

Хорошо:

```text
OpenCode, найди все упоминания `oldName` и верни список файлов. Ничего не меняй.
```

Хорошо:

```text
OpenCode, замени `oldName` на `newName` только в docs/ai. После этого покажи diff.
```

Плохо:

```text
OpenCode, наведи порядок во всём проекте.
```

## Отчёт OpenCode

После выполнения OpenCode должен вернуть:

- что сделал;
- какие файлы затронул;
- какие команды запускал;
- какие ошибки были;
- что не проверялось.

## Безопасность

OpenCode не должен скрывать неудачные команды. Если команда не запускалась или упала, это нужно прямо указать.
