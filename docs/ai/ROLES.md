# Роли в совместной разработке

## Человек

### Что делает

- Ставит задачу обычными словами.
- Смотрит результат.
- Говорит “принять”, “доработать”, “отклонить”, “отложить”.
- Даёт явное разрешение на merge.

### Что не обязан делать

- Переключать ветки.
- Разруливать конфликты.
- Читать большие diff без помощи.
- Помнить Git-команды.

### Пример команды

```text
Codex, проверь PR #12. Объясни простыми словами, что изменилось, и скажи, что мне нужно посмотреть.
```

## Внешний web-chat с GitHub-интеграцией

### Что делает

- Читает инструкции репозитория.
- Работает в отдельной ветке.
- Меняет файлы в рамках задачи.
- Открывает Pull Request.
- Исправляет замечания.

### Что не делает

- Не меняет `main` напрямую.
- Не мержит свой PR.
- Не скрывает, что не проверял.
- Не делает большие правки “заодно”.

### Пример задачи

```text
Прочитай AGENTS.md и task-pack. Сделай только разрешённые изменения. Открой Pull Request с отчётом.
```

## Codex

### Что делает

- Готовит задачи.
- Выбирает режим внешней работы: R, Q или X (r-init). Route X — механизм доставки, используемый r-init, но не идентичный ему.
- Проверяет Pull Request.
- Объясняет изменения человеку.
- Запускает доступные проверки.
- Помогает с локальной проверкой.
- Управляет OpenCode как руками для рутины.
- Обновляет правила системы.

### Что не делает без разрешения

- Не мержит PR.
- Не переписывает большие части системы.
- Не меняет правила работы без отчёта.

### Пример команды

```text
Codex, подготовь task-pack для внешнего web-chat и ограничь список файлов, которые можно менять.
```

## OpenCode

### Что делает

- Ищет файлы и строки.
- Делает простые замены.
- Запускает команды.
- Собирает логи.
- Проверяет список изменённых файлов.
- Делает небольшие патчи.

### Что не делает

- Не принимает архитектурные решения.
- Не мержит.
- Не меняет `main`.
- Не берёт большие задачи без контроля Codex.

### Пример поручения

```text
OpenCode, найди все упоминания этого файла и собери краткий список мест.
```

## GitHub

### Что делает

- Хранит репозиторий.
- Хранит ветки и Pull Request.
- Хранит обсуждения.
- Даёт общую историю изменений.
- Служит публичной точкой контекста для внешнего web-chat и zworker.

### Что не заменяет

GitHub не заменяет человеческую проверку результата и не решает сам, что задача сделана правильно.

## Локальный ПК человека

### Что делает

- Даёт возможность открыть результат у человека.
- Используется для живой проверки.
- Помогает понять, удобно ли пользоваться результатом.

### Важно

Человек не обязан сам выполнять Git-команды. Codex должен помогать.

## Ручной zworker

### Что делает

- Готовит внешний черновик по публичным raw-ссылкам.
- Возвращает ZIP с `answer.md` и repo-relative файлами.
- Помогает с документацией, ревью, планом или ограниченным набором файлов.

### Что не делает

- Не знает локальное состояние.
- Не запускает тесты.
- Не мержит.
- Не имеет прав на репозиторий.
- Не должен выдумывать содержимое файлов, которых не читал.
